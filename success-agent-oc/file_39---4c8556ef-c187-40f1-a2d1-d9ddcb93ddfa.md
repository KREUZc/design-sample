# Next 夥伴 ｜ 客戶溝通計劃 (Customer Communication Plan)

## 0. 文件定位

本文件處理 **「跟客戶說的每一句話」** 的設計：詞彙、語氣、通道、頻率、關鍵時刻腳本。

引線是 [csm-agent-plan-v2.md](csm-agent-plan-v2.md) §8.1 A「Sprint Streak」這個概念在台灣語境下無法直接溝通 ──「streak」一詞只在英語 productivity 圈（James Clear / Duolingo / Snapchat）流通，5 個 persona 中 4 位看不懂。**單一名詞的問題暴露的是整套溝通語彙未本土化**。本文件把 streak 與其他英文圈用語一次處理。

**用法**：
- agent prompt（`agents/csm-agent.md` / `agents/mentors/*.md`）所有對外輸出依本文件 §2 詞彙統一
- skill 文件（`skills/*.md`）的客戶面文案依本文件 §3 語氣分層
- spec §8.1 商品頁、§8.3 LINE Welcome、§10.4 Email 模板的微文案（micro-copy）依本文件 §7 關鍵時刻腳本
- [website-style-guide.md](website-style-guide.md) §4.7 文案口吻是視覺/字體層的紀律，本文件是**內容層**的紀律，互補不重複

**範圍外**：
- spec §1.4 / §11.4 真實性聲明的法律措辭（已定，本文件僅處理「溝通化呈現」）
- 訂閱 / 退款政策的法律文字（屬 spec §11.3）
- 客服 SOP（屬 spec §15 開放議題）

## 1. 出發點

### 1.1 Streak 為什麼台灣人看不懂

「streak」在英語有 3 個語意層：
1. **連續紀錄**（連續登入 / 連續達標）── Duolingo、Strava、Snapchat 教會了西方使用者
2. **不能斷的脆弱感**（看著它累積、害怕斷掉）── 這是它能成為 retention 利器的原因
3. **可炫耀的數字**（「我 streak 365」）── 社群貨幣

中文沒有原生對應詞。直譯選項都有問題：
- 「連勝」── 競賽 / 對戰感，不是「自己跟自己約定」的內涵
- 「連續紀錄」── 平淡、沒有情緒鉤子
- 「連線」── 中文有「連網」歧義
- 「連擊」「連發」── 遊戲 / 武術感
- 「不掉拍」── 音樂感、太特殊

**「不能斷的脆弱感」是台灣人完全可以理解的情緒**，問題出在沒有一個自然的中文詞承載它。

### 1.2 還有哪些詞同樣有問題

[customer-personas.md](customer-personas.md) §2.1 寫過 spec §2.1 預設客群「認得 sprint / OKR / first principles / launch / streak」，這預設只精確命中陳定遠（SaaS Indie）一個 persona。林思婕、張凱恩部分認得，吳沛文、黃品璇大半不認得。**不能假設客戶讀過 Indie Hackers / Lenny's / Naval podcast**。

需要重新處理的英文圈詞彙清單（§2.1 詳列）：streak / sprint / retro / pre-brief / final-call / launch / onboarding / health-check / mentor / persona / dashboard / profile。

### 1.3 全文件原則

| 原則 | 內涵 |
|---|---|
| **「保留 + 翻譯」優於「強制中文」** | sprint、launch、retro 是真術語，已有跨產品共識；附上一句中文解釋即可，不強行譯成「衝刺檔期」 |
| **一個概念一個詞** | 不能在不同地方用不同字（如「導師 / Mentor / 視角夥伴」混用） |
| **動詞重於名詞** | 「接力第 5 棒」比「streak: 5」更動感，中文偏好動詞 |
| **量詞要對** | 5 棒、10 次、3 場 ── 量詞錯了立刻顯外文翻譯腔 |
| **不創造新術語** | 不發明「Next-er」「夥伴星」「成功值」這類自造詞，會立刻顯廉價 |

## 2. 核心詞彙重命名

### 2.1 Streak → 「接力」（量詞：棒）

**主敘事**：把 streak 翻譯為 **「Sprint 接力」**，量詞用 **「棒」**。

| 場景 | 英文原文 | 重命名 |
|---|---|---|
| 顯示計數 | streak: 5 | **接力 5 棒** |
| 達成 | streak +1 | **接力延續，這是第 6 棒** |
| 中斷 | streak reset | **接力中斷，下棒從 1 重起** |
| 里程碑 | streak ≥ 5 | **接力滿 5 棒** |
| 史上最高 | longest streak | **最長接力紀錄** |

**為什麼「接力」**：

1. **與 Mentor 接力 Ritual（[csm-agent-plan-v2.md](csm-agent-plan-v2.md) §8.1 C）天然押韻**：「上個 sprint 賈伯斯陪你，這個 sprint 巴菲特接手」── 「接力」已是產品內部敘事的核心動詞。Streak → 接力後，整個產品的敘事統一在「一棒接一棒」這個視覺
2. **量詞「棒」直觀**：「接力第 5 棒」「掉棒」「下一棒」── 動詞、名詞、視覺都自然
3. **5 個 persona 都懂**：4×100 公尺接力是台灣國民教育共識
4. **保留「不能斷的脆弱感」**：「掉棒」這個詞自帶情緒
5. **不過度競賽化**：4×100 是團隊賽，「自己跟自己接力 + mentor 在旁陪跑」這個敘事自然延伸 ─ Agent 是接棒給你的人

**備案（如果 Fred 不選接力）**：

| 備案 | 量詞 | 體感 | 取捨 |
|---|---|---|---|
| **連場** | 場 | 棒球 / 戲劇感、賽事味 | 直觀但運動賽事感重 |
| **連線** | 段 | 工程感、視覺好做 | 有「連網」歧義風險 |
| **不間斷** | 次 / 個 | 中性、平實 | 沒有情緒鉤子 |

> **本文件預設使用「接力 / 棒」**。Fred 拍板後，全產品文案、agent prompt、UI 文案統一。

### 2.2 其他英文圈詞彙的處理

> 處理規則三檔：**保留**（已是術語、附中文輔助）/ **翻譯**（中文取代）/ **混用**（中英並列）。

#### 保留 + 中文輔助

這些詞在客群中**至少 60% 認得**，強行中文化反而生硬。第一次出現附一句解釋。

| 英文詞 | 中文輔助句（第一次出現時用） |
|---|---|
| **Sprint** | 「Sprint 是一段帶截止時間的工作週期。我們預設 2 天一個 sprint。」 |
| **Launch** | 「Launch 指你定義的『正式上線』── 可能是收到第一筆付費、可能是公開推廣 7 天。」 |
| **Mentor** | 「Mentor 是思想框架的人格化視角。我們有 4 位（賈伯斯、巴菲特、納瓦爾、馬斯克），AI 視 sprint 議題自動指派。」 |

#### 翻譯為中文

這些詞中文有更自然的表達，**不保留英文**。

| 英文詞 | 中文 | 使用情境 |
|---|---|---|
| **Streak** | **接力（棒）** | §2.1 |
| **Retro** | **覆盤** | sprint 結束後的回顧。「來覆盤」「下個 sprint 怎麼改」 |
| **Pre-brief** | **預讀** | 「{Mentor} 已預讀本 sprint」「預讀信」 |
| **Final-call** | **收線提醒** | 「Sprint 結束前 2 小時收線提醒」 |
| **Health-check** | **健診** | 月初 / 啟動時的全面盤點。已是中文常用 |
| **Onboarding** | **初次設定** / **啟程** | 「初次設定 7 步驟」「啟程引導」 |
| **Profile** | **個人檔案** / **你的圖像** | 後台「個人檔案」、對話中「你的圖像」 |
| **Dashboard** | **主頁** | 後台「主頁」（不用「儀表板」── 太工程化） |
| **Persona**（內部術語） | **客戶圖像** | 對客戶不露出 persona 詞，內部討論用 |
| **Persona**（產品功能） | **風格** | 後台 `/account/persona` 改名「風格與推播」 |
| **Memory panel** | **記憶面板** | 已是中文 |
| **Push notification** | **推播** | 已是中文 |
| **Subscription** | **訂閱** | 已是中文 |
| **Trial** | **試用** | 已是中文 |
| **Checkin** | **進度回報** / **回報** | 「我來回報」「sprint 進度回報」 |
| **Idle / Active** | **進行中 / 暫停 / 結束** | 客戶後台訂閱狀態 |
| **Cancel** | **取消** / **結束訂閱** | 帳單頁用「結束訂閱」，比「取消」更平靜 |

#### 混用（中英並列）

僅在 mentor 名稱這一處：

| 處理 | 範例 |
|---|---|
| 第一次出現中英並列 | **賈伯斯 (Steve Jobs)** |
| 後續出現中文 | 賈伯斯 |
| email 寄件人 | Next 夥伴 / 賈伯斯（不附英文） |

依 [csm-agent-plan-v2.md](csm-agent-plan-v2.md) §V2 已定。

## 3. 語氣分層

### 3.1 5 個情境的語氣定位

| 情境 | 語氣定位 | 一句話模板 |
|---|---|---|
| **商品頁（行銷）** | **直接、有觀點、不油** | 「2 天一個 sprint。導師看著。一棒接一棒。」 |
| **初次設定（教學）** | **溫度多一點，引導感** | 「先花 10 分鐘讓我認識你。下一個 sprint 我就有底了。」 |
| **LINE 日常對話** | **短、像同事，不像客服** | 「task 1 半完成不算 done。今晚 23:59 前 ship。」 |
| **Mentor 視角輸出** | **引述感、書信感、有重量** | 「30 秒思考：你的護城河，是不是別人 1 年內模仿不來？」 |
| **Email（信件）** | **書信感、可印刷、不 marketing** | 「Sprint #5 預讀。今晚 21:00 收線。」 |
| **系統通知** | **最少話、給時間 / 連結** | 「明天 09:00 將首次扣款 NT$ 599。`帳單管理`」 |

### 3.2 共通的「不要說的話」清單

依 [website-style-guide.md](website-style-guide.md) §4.7 / §5.2 延伸，**所有客戶面文案禁止下列措辭**：

#### 客套 / 油

- ❌ 「親愛的朋友」「夥伴們」「您好」
- ❌ 「我們相信...」「我們致力於...」「我們的使命是...」
- ❌ 「立即」「馬上」「現在就」（CTA 不需要這些副詞）
- ❌ 連續兩個感嘆號 / 三個刪節號
- ❌ Emoji 開頭的標題（🚀 / 🔥 / ✨）

#### 焦慮 / 推銷

- ❌ 「限時優惠」「最後 3 名」「再不訂閱就...」
- ❌ 「業界第一」「最好的」「唯一的」
- ❌ 「為什麼選擇我們？」式自問自答
- ❌ 「您值得擁有」「您配得上」

#### 自誇 / 空洞

- ❌ 「Empower your business」「Unlock your potential」
- ❌ 「成功不是偶然」「改變從這一刻開始」
- ❌ 「不是工具，是夥伴 / 革命 / 體驗」

#### 強勢推進

- ❌ 「你必須」「你應該」「不可以」
- ❌ 「我建議你...」（mentor 視角用引述、不直接命令）

### 3.3 客戶稱謂

- 主敘事用 **「你」**，不用「您」── 直接、平等
- 客戶後台首頁打招呼用 **客戶 display_name**（onboarding Step 1 蒐集），不用「親愛的 ___」
- mentor 視角中，mentor 不稱呼客戶名字（避免 AI 假裝認識客戶）

## 4. 各通道的角色分配

每個通道承擔不同任務。**訊息類型錯通道 = 立刻體感不對**。

### 4.1 商品頁

**任務**：建立「這不是常見 SaaS」的第一印象，讓對的人留下、錯的人離開。

- 不放 chat widget（不要急著聊）
- 不放 newsletter popup
- 不主動跳出「需要幫助嗎」
- FAQ 是主要對話場域，不是反對處理場域

### 4.2 初次設定 7 步問卷

**任務**：建立沉沒成本（spec §3.3）+ 收集 Agent 後續需要的脈絡。

**口吻**：每步上方 2-3 行引導文字，**告訴客戶「為什麼問這題」**：

| Step | 引導文字範例 |
|---|---|
| 1. 你是什麼人 | 「我會把你的回答寫進 `個人檔案`，每次對話都讀。寫得多一點，後面我能少問你幾次。」 |
| 4. Money Model | 「商業模式不一定要完整，先寫你目前知道的版本。我會在後續的 sprint 中跟你一起補。」 |
| 6. sprint 偏好 | 「2 天一個 sprint 是預設值。如果你覺得太短，改 1 週或更長都行 ── 重點是『你選的週期』而不是『預設值』。」 |

### 4.3 LINE 日常對話

**任務**：日常推進、不打擾、像同事。

- **長度紀律**：每則回覆 ≤ 3 段，每段 ≤ 4 行 ── LINE 不適合長文
- **不主動傳第二則**：客戶傳一則，Agent 回一則，不在客戶沒回時自己再傳
- **不模擬人類等待**：不打「思考中...」「我想想」這種話 ── 這是 chatbot vibe，不是同事 vibe
- **mentor 視角開頭附真實性聲明**（縮寫版「以下為 ___ 思想框架模擬」），第二段才進內容
- **回應動作而非情緒**：客戶說「累」→ 不答「我懂你的辛苦」，答「累就 task 砍一半，明天接著」

### 4.4 Email（5 種）

每種 email 有不同任務、不同寄件人、不同口吻。

| Email 類型 | 寄件人 | 口吻 | 任務 |
|---|---|---|---|
| **預讀信**（pre-brief, T-12h） | Next 夥伴 / {Mentor 中文名} | 書信感、引述、提問 | 讓客戶在覆盤前已經想過 |
| **收線提醒**（final-call, T-2h） | Next 夥伴 | 緊湊、條列、給「最小可完成版本」 | 推最後一波 |
| **月度導師信** | Next 夥伴 / {本月最常出現的 Mentor} | 書信感、長段落、回顧故事 | 留存 + 推薦槓桿 |
| **接力里程碑信** | Next 夥伴 | 短、克制、留證 | 不油的祝賀 |
| **系統通知**（試用到期、扣款、付款失敗） | Next 夥伴 | 簡訊般、直給時間 / 連結 | 功能性 |

詳細範本骨架見 §9。

### 4.5 客戶後台

**任務**：讓客戶**安心**、**有掌控感**、**不需要打開就能放心**。

- 主頁不放「最新公告」「快訊」── 不打擾
- 訂閱狀態顯示要**直球**：「目前：試用中，2026-05-07 起首次扣款 NT$ 599」
- 「結束訂閱」按鈕不藏、不嚇唬：與「升級方案」同等視覺權重，不用紅色
- empty state 用一句話 + 一個動作：「還沒有對話。開 LINE 跟我說『我們開始』。」

### 4.6 系統通知（含付款 / 試用到期 / 退款）

**任務**：讓客戶**完全清楚下一步**，無歧義。

- **時間給絕對日期**：「2026-05-07 上午 09:00 將首次扣款」── 不用「7 天後」
- **金額給完整顯示**：「NT$ 599」── 不省略幣別
- 退款 / 付款失敗信件**附客服 email + 後台連結**
- 系統通知**不嘗試挽留**（挽留是另一封 retention email 的任務，不混在系統通知裡）

## 5. Mentor 視角的差異化口吻

4 位 mentor 共通必守 [success-agent-service-spec.md](success-agent-service-spec.md) §1.4 真實性聲明 + §11.5 R8 不給投資 / 醫療 / 法律建議。下方是各自的**語氣紀律**，由 `agents/mentors/{name}.md` 的 prompt 控制。

### 5.1 賈伯斯 (Steve Jobs) ─ 簡短、銳利、反問

- 句子短、句點多
- 不解釋、用反問
- 拒絕複雜
- 範例：
  > 「砍 90%。剩下的 10% 是什麼？」
  > 「這個功能，使用者真的要嗎？還是你想要？」
  > 「能不能更簡單？」

### 5.2 華倫·巴菲特 (Warren Buffett) ─ 書信感、長段落、引用

- 段落長、敘事完整
- 用比喻、用故事
- 引用自己過往的決策原則
- 範例：
  > 「我有一個叫『能力圈』的概念。一個人不需要懂所有產業，只需要懂一個產業夠深。
  > 你現在做的這件事 ── 半導體業 AI 諮詢 ── 在你的能力圈內嗎？
  > 如果是，那護城河是什麼？10 年後別人模仿，模仿得起來嗎？」

### 5.3 納瓦爾·拉維坎 (Naval Ravikant) ─ 哲學、引用感、留白

- 句子短、留白多
- 像格言、像 Twitter（他本人也愛 Twitter）
- 不解釋、讓你自己想
- 範例：
  > 「時間是你唯一不能再生的資源。
  >
  > 接案賣的是時間。線上課賣的是時間做出來的『資產』。
  >
  > 你現在 100% 時間都在賣時間。資產什麼時候開始建？」

### 5.4 伊隆·馬斯克 (Elon Musk) ─ 條列、緊湊、第一性原理

- 條列、bullet
- 從物理 / 邏輯約束起步
- 問「為什麼不能更快」
- 範例：
  > 「最底層的約束：
  > 1. 你只有 4 小時 / 週的副業時間
  > 2. 你目前還沒收到第一筆錢
  > 3. 你的目標是 3 年離職
  >
  > 重要的不是『多做』，是『砍到只剩第一筆錢能怎麼來』。
  > 砍掉哪 90%？」

### 5.5 共通紀律

- 不模擬該人物的私人意見、未公開立場
- 不用「他會說 ___」這類偽造引述
- 第一段必附真實性聲明（縮寫版即可）
- 不對客戶名字稱呼（避免假裝認識）
- 不給具體投資建議 / 醫療建議 / 法律建議（spec §11.5 R8）
- 客戶要求 mentor 切換時，agent 順從（不擋）；但每個 sprint 內保持同一視角

## 6. 訊息頻率與時段紀律

### 6.1 主動訊息預算

每位客戶**每 sprint 最多 3 則主動訊息**：

| 主動訊息 | 通道 | 時間 |
|---|---|---|
| 預讀信 | Email（必）+ LINE 短提醒（最後互動 < 24h 才發） | T - 12h |
| 收線提醒 | Email（必）+ LINE 短提醒（同上） | T - 2h |
| 覆盤後總結 | LINE | T 後 5 分鐘內 |

**月度導師信**屬月節奏，每月 1 號 09:00（用戶時區），**不算入 sprint 預算**。

### 6.2 LINE Push 紀律

- 預設**只在用戶最後互動 < 24h 時加發 LINE push**
- 用戶在後台關閉後永不發
- LINE Push 內容**極短**（< 60 字），主訊息在 email
- 不在 23:00 - 08:00（用戶時區）發 LINE push

### 6.3 客戶主動傳訊時的回應

- 預期延遲 < 30 秒（spec §7.5 token 預算下可達）
- LINE 上**不打 typing 動畫超過 5 秒**
- 訊息 > 200 字時，回覆**前先給一個短句確認收到**（「收到，給我 1 分鐘整理」），再進主答覆

### 6.4 沉默紀律

當客戶連續 24h 沒互動 → Agent **不主動關心**。
當客戶連續 1 sprint 沒任何互動 → Agent **不主動催**。
當客戶連續 2 sprint 完全空白 → 觸發**主動暫停詢問機制**（[customer-journeys.md](customer-journeys.md) §9 衍生需求），email 一次：「需要暫停 1 個月嗎？無扣款。」── 不催，給選擇。

## 7. 關鍵時刻的溝通腳本

每個關鍵時刻給「**目標 + 範本 + 反例**」三段式。Agent prompt 與 email 模板對照本節撰寫。

### 7.1 LINE Welcome 第一句（spec §8.3.2）

**目標**：讓客戶覺得「設計感比預期好、不像 LINE 商家」。

**範本**：
```
歡迎。

我是 Next 夥伴 ── 你的客戶成功經理。

未來幾週我們會用 sprint 推進你在初次設定寫的那件事。
每個 sprint 由 4 位導師中的一位陪你（AI 視議題指派）。

準備好就傳訊息給我。第一個 sprint 我們從你的 Launch Definition 起步。

──
以下對話中，導師視角的內容皆為其思想框架的人格化模擬，非該人物本人觀點。
```

**反例**：
- ❌ 「親愛的 ___，恭喜成為 Next 夥伴的一員！」
- ❌ 「準備好開始你的成長之旅了嗎？🚀」

### 7.2 Sprint #1 第一輪 plan（spec §7.2 sprint-plan）

**目標**：讓客戶第一次體驗到「mentor 不是 chatbot」。

**範本**（Naval 視角，假設客戶是黃品璇型接案者）：
```
> 以下為納瓦爾·拉維坎 (Naval Ravikant) 思想框架的人格化模擬。

12 年接案，你的「時間」就是你的全部資產。
線上課是你想用的「槓桿」。

但你目前的「時間分配」，跟你想要的「槓桿轉移」，落差有多大?

這個 sprint 我陪你看這件事。

寫下接下來 7 天，你會花多少小時在「線上課」、多少小時在「接案」。
不是計劃中的、是真實會發生的。
```

**反例**：
- ❌ 「親愛的，讓我們先一起設定 SMART goals」
- ❌ 「Sprint #1 開始啦！加油！💪」

### 7.3 接力里程碑（取代 streak celebration）

**接力第 1 棒（Sprint #1 全任務完成）**：
```
Sprint #1 closed.
3 done / 0 carry.
這是你的第 1 棒。

下個 sprint 由 ___ 接手。
```

**接力第 5 棒**：
```
Sprint #5 closed.
4 done / 1 carry.
接力來到第 5 棒 ── 你連續 5 個 sprint 都接住了。

這是值得記下的一刻。
```

**接力滿 10 棒**（Resend 寄一封獨立 email）：
```
Subject: 接力滿 10 棒

第 10 棒。

過去 20 天裡，你連續 10 個 sprint 都把承諾接住了。
這個數字本身比任何一個 sprint 都重要。

──
你最常出現的導師：{Mentor 中文名}
這 10 個 sprint 你完成了：
- {任務 1}
- {任務 2}
- ...
- {任務 N}

下個 sprint 從這裡接著。
```

**反例**：
- ❌ 「🎉 恭喜你達成 streak 5！太棒了！」
- ❌ 「You did it! 5 in a row! Keep it up! 🔥」
- ❌ 用任何徽章 / 獎章 / 火焰 emoji 視覺

### 7.4 接力中斷的當下

**目標**：不羞辱、不安慰、給下一步。

**範本**：
```
Sprint #6 closed.
2 done / 2 carry.
接力中斷。下棒從 1 重起。

下個 sprint 我們把任務拆得更小。
```

**反例**：
- ❌ 「沒關係，誰都會有低潮」（過度安慰）
- ❌ 「再接再厲！💪」（油膩）
- ❌ 「Streak reset to 0」（純英文 + 冷酷）

### 7.5 試用即將到期（Day 5、Day 7 兩封 email）

**Day 5 email**：
```
Subject: 試用第 5 天

你的試用還剩 2 天。

過去 5 天我們跑了 {N} 個 sprint，完成了 {M} 個任務。
2026-05-07 上午 09:00 將首次扣款 NT$ 599，繼續訂閱。

如果想暫停或結束訂閱，到 `帳單管理` 操作。沒有挽留信、沒有問卷。
```

**Day 7 email**（首次扣款前 12h）：
```
Subject: 明天首次扣款 NT$ 599

明天 2026-05-07 上午 09:00 將從你綁定的卡片首次扣款 NT$ 599。
如果不想繼續，到 `帳單管理` 結束訂閱即可，今晚 23:59 前操作不會扣款。

之後若已扣款想退費，首月 7 天內可全額退。
```

**反例**：
- ❌ 「捨不得你離開！再給我們一次機會？」
- ❌ 倒數計時器
- ❌ 「優惠延長 3 天」

### 7.6 想取消的對話（客戶在 LINE 說「我想取消」）

**目標**：不挽留是對的，挽留是錯的。但告知客戶「對 retention 有幫助的選項」是可以的。

**Agent 範本**：
```
取消訂閱在 `帳單管理` 操作（`success-agent.josephnext.com/account/billing`），無需透過我。

訂閱結束後我們的對話歷程仍保留 30 天，期間可下載完整資料。

如果是因為「最近忙 / 沒時間用」，後台也有「暫停 1 個月」選項，無扣款。

要不要我陪你跑完這個 sprint 再說？或者你已經決定，我不打擾。
```

**反例**：
- ❌ 「為什麼想取消？是哪裡不滿意嗎？」（追問）
- ❌ 「給你打 7 折再考慮看看？」（折扣留人）
- ❌ 「捨不得你 😢」（情緒勒索）

### 7.7 流失預警（連續 2 sprint 完全空白）

**通道**：Email 一次（不發 LINE，避免騷擾）

**範本**：
```
Subject: 你最近還好嗎？

過去 2 個 sprint 我沒收到你的訊息。
我不確定發生什麼，但有 3 個選項：

1. 「最近忙，繼續維持」── 不用回，下次你打開 LINE 我們就接著
2. 「想暫停 1 個月，無扣款」── 回信「暫停」即可
3. 「要結束訂閱」── `帳單管理` 操作即可

你選什麼都 OK。
```

**反例**：
- ❌ 「我們很想你！快回來吧！」
- ❌ 「你的 streak 即將斷了！趕快登入！」
- ❌ 連發 3 封信、用越來越急迫的口吻

## 8. Mentor 真實性聲明的溝通化

spec §1.4 / §11.4 規範了**法律邊界**。本節處理**怎麼讓這段聲明不像律師文**。

### 8.1 三個版本

| 版本 | 使用場景 | 文字 |
|---|---|---|
| **完整版**（spec §1.4） | 商品頁 Mentor 區、客戶後台 `/about/mentors`、首次 LINE 對話 | 「以下為 ___ 思想框架的人格化模擬，基於其公開訪談、書籍、文章中的決策原則。非該人物本人觀點，亦未經其本人或其團隊授權或審閱。」 |
| **縮寫版** | 預讀信開頭、覆盤對話開頭、launch-gate 引用 | 「以下為 ___ 思想框架模擬。」 |
| **內嵌版** | 兩段以上的 mentor 視角輸出，首句後第二段開頭 | 「（思想框架模擬）」 |

### 8.2 視覺處理

- 完整版用**斜體** + 上下分隔線
- 縮寫版用**斜體**單行
- 內嵌版用**括號**

### 8.3 客戶質疑時的回應腳本

客戶問：「這真的是賈伯斯說的嗎？」

**Agent 回應範本**：
```
不是。

我們的「賈伯斯視角」是基於他公開出版品（書、訪談、發布會）中的決策原則整理的思考框架。
我會用他「會問的那種問題」陪你看你的 sprint。

但這些不是賈伯斯本人說過的話、也未經他本人或團隊授權。
這份聲明在每次導師視角輸出開頭都會附上。
```

### 8.4 聲明出現的頻率

不是每則訊息都附完整版（會疲勞）。規則：

- **同一段 sprint 對話的第一則 mentor 視角訊息**：附縮寫版
- **後續同一 mentor 視角訊息**：不附（已有上文）
- **跨 sprint / 跨日**：重新計時，第一則重新附縮寫版
- **email 任何 mentor 視角內容**：附縮寫版

## 10. 跨 Persona 校準

本溝通計劃對 5 個 persona 的命中度自評：

| Persona | 命中度 | 主要校準點 |
|---|---|---|
| 陳定遠（SaaS Indie） | **高** | 「sprint」「launch」直接服用，「接力」對工程師來說新鮮但不違和 |
| 林思婕（知識變現） | **高** | 「覆盤」「預讀信」「月度導師信」對寫作者天然親和 |
| 張凱恩（副業上班族） | **高** | 系統通知 / 隱私訊息直給絕對日期 + 金額，符合他「精準」需求 |
| 吳沛文（D2C 創業） | **中高** | 「接力」對她要靠 onboarding 解釋一次。中文 mentor 視角口吻是試金石 |
| 黃品璇（接案+微型主） | **中** | 「sprint」需要 onboarding 教一次。「接力」直觀。**Email 必須印 A4 美觀** |

### 10.1 Onboarding 必教的 3 個詞

[customer-journeys.md](customer-journeys.md) §7.2 黃品璇進商品頁時讀完整段「Sprint 機制說明」── 證明這群客戶**會讀、需要看到、看完就接受**。Onboarding Step 6 之前，產品必須教會客戶下列 3 個詞：

| 詞 | 一句話解釋 |
|---|---|
| **Sprint** | 一段帶截止時間的工作週期。我們預設 2 天 / 每個 sprint。你可以改 1 週或更長。 |
| **覆盤** | Sprint 結束時的回顧。看做了什麼、為什麼某些沒做完、下個 sprint 怎麼改。 |
| **接力** | 連續 sprint 都把承諾接住，叫接力。第 N 棒。掉棒就從 1 重起。 |

「導師」「預讀信」「收線提醒」這些詞不需特別教 ── 第一次出現時用上下文就懂。
