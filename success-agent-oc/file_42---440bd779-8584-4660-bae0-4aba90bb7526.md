# Next 夥伴 ｜ 網站風格指南 (Website Style Guide)

## 0. 文件定位

本文件是 **給設計團隊（claude design / 視覺設計師）的策略 brief**，依據 [customer-personas.md](customer-personas.md) 5 個 persona 與 [customer-journeys.md](customer-journeys.md) 的旅程觀察，鎖定 **核心視覺命題**、**情緒方向**、**反例邊界**，留下風格的「為什麼」與「不能是什麼」。

**用法**：
- claude design 接手後依本指南展開：色彩具體值、字體選擇、組件規格、wireframe
- 與 [success-agent-service-spec.md](success-agent-service-spec.md) §8.1 商品頁、§8.3 LINE Welcome、§8.4 客戶後台、§10.4 Resend 模板對齊
- 本文件**不指定 hex / px / 字體名稱**；只給語意、感受、反例、人物 / 場域 / 物件參照

**範圍外**：
- 元件層級的設計（按鈕、表單、卡片）
- design token 命名與結構
- 響應式斷點規格
- Figma 檔案結構

## 1. 核心命題：「成功後的形象想像」是什麼

### 1.1 不是

- ❌ **不是奢華感**（gold accent、深色 + 金色文字、皮革紋理）── 那是 luxury watch / private banking 的語言，不是 knowledge worker 的語言
- ❌ **不是科技未來感**（霓虹、漸層、glow、賽博）── 那是 web3 / crypto / 新創 hype 的語言
- ❌ **不是 SaaS template 的「Hello World」**（紫色漸層 hero、illustration 帶笑臉、「Empower Your Business」）── 那是「沒個性的工具」的語言
- ❌ **不是中文知識付費的「成功學」感**（金句卡 + 大頭照 + 紅色加粗）── 那是 LINE 群轉發的語言

### 1.2 是

> **「客戶 5 年後成為他想成為的那個人」── 這個產品在那個時候會出現在他的書桌、他的瀏覽器、他的截圖。**

把這句話翻成設計可執行的 4 個原則：

| 原則 | 內涵 | 設計含義 |
|---|---|---|
| **被尊重的精準感** | 用詞正確、不被當小孩、不被推銷 | 文案密度低 / CTA 不嗲 / 不用驚嘆號 |
| **時間感** | 看起來經得起 5 年、不追潮流 | 不用當下流行的 illustration 風格、不用會過時的漸層 |
| **靜默的厚度** | 留白多、字重對比強、訊息少而準 | 一頁不擠 8 個區塊、首屏不超過 3 句話 |
| **人感** | 不像 SaaS template、像「某個人在認真做這件事」 | 視覺有「主理人指紋」、不用 stock photo、文案有觀點 |

### 1.3 一句話風格定位

> **「成熟知識工作者的書桌」── 安靜、有重量、有他自己的東西。**

claude design 在所有設計決策卡關時，回到這句話判斷。

## 2. 5 個 Persona 對「成功」的視覺想像

每個 persona 對「成功的我」有不同畫面。設計策略不是 5 個都討好，而是 **找出 5 種想像的交集，作為主軸；差異作為次要層的微調**。

### 2.1 陳定遠（SaaS Indie 32 男）─ 開發者尊嚴

**他想成為**：Pieter Levels / Justin Welsh / Lenny Rachitsky / Tony Dinh ── 獨立、可被驗證的成績、不油的存在。

**視覺想像**：
- 場域：自己的書桌、雙螢幕、終端機在跑、植物一盆
- 工具參照：**Stripe / Linear / Vercel / Cursor / Raycast** ── 開發者級別的精準
- 字體：等寬字 / sans-serif neo-grotesque、不矯飾
- 色彩：低彩度、單一強調色（通常是該品牌的 signature）
- 排版：充分留白、grid 嚴謹
- 厭惡：illustration 帶笑臉、漸層、emoji 充斥

**他成功時的截圖**：他發了一條 Twitter「Project C 收到第一個付費客戶」，配圖是 Stripe dashboard 截圖。Next 夥伴的 mentor 對話被他並排截在旁邊。**設計含義**：產品 UI 必須**截圖好看**，不會破壞他的個人品牌敘事。

### 2.2 林思婕（知識變現 36 女）─ 作者權威 + 溫度

**她想成為**：Justin Welsh（中性）/ Brené Brown / Marie Forleo / Anne Helen Petersen / 朱騏 ── 有自己思想、被讀者長期追隨、不是網紅而是作者。

**視覺想像**：
- 場域：書架、寫作桌、咖啡杯、紙筆共存
- 工具參照：**Substack / Ghost / Notion / Readwise** ── 寫作者級別的克制
- 字體：襯線體（serif）作為閱讀字、寬鬆行距、左對齊不置中
- 色彩：象牙白底 / 深墨色字 / 一點點暖色（不是粉紅）
- 排版：長段落、留白如書頁
- 厭惡：銷售感重、計時器 / 倒數、「成為自己」金句卡

**她成功時的截圖**：她在 Substack 發了一篇「我用 AI mentor 一年的觀察」，配圖是月度 mentor 信的版面。**設計含義**：Email 模板必須有「書信感」，不是 marketing email。

### 2.3 張凱恩（副業上班族 38 男）─ 顧問沉穩

**他想成為**：Stratechery 作者 Ben Thompson / Reforge 顧問 / a16z partner / 中年版本的自己（離職後做顧問）── 可信、有累積、不浮誇。

**視覺想像**：
- 場域：辦公桌雙螢幕、白板、深色木紋
- 工具參照：**Notion enterprise / Stratechery / HBR / Bloomberg Terminal** ── B2B 顧問業的沉穩
- 字體：新聞 / 學術風 ── serif 標題 + sans-serif 內文
- 色彩：低飽和深色、深藍 / 深綠 / 焦糖
- 排版：資訊密度可以略高（他習慣讀長文 + 表格）
- 厭惡：illustration、emoji、「年輕新創 vibe」

**他成功時的截圖**：他不會公開截圖（怕公司知道）。但他的私人筆記裡會貼 Buffett pre-brief email。**設計含義**：Email 內容必須**單獨閱讀有價值**，不依賴上下文。

### 2.4 吳沛文（D2C 創業 27 女）─ 品牌品味

**她想成為**：Glossier 創辦人 Emily Weiss / Aesop 主理人 / Sun Day Body / Topicals 品牌團隊 ── 有品味的女性創業者，作品被同業尊敬、被媒體報導。

**視覺想像**：
- 場域：白牆 + 植物 + 自家產品照、網頁設計乾淨
- 工具參照：**Aesop website / Glossier / Sun Day / Verve Coffee / Apartamento Magazine** ── 品牌精緻度的天花板
- 字體：細襯線標題 + 寬鬆無襯線內文
- 色彩：米白、灰黑、自然色（莫蘭迪不是甜膩 pastel）
- 排版：構圖意識強、留白即奢侈
- 厭惡：「便宜的精緻感」（廉價金邊、AI 生成感、stock photo 笑臉女生）

**她成功時的截圖**：她在 IG 發限動「這是我每天早上看的東西」，配的是 Next 夥伴 LINE 對話 + 自家咖啡杯。**設計含義**：LINE Flex Message 必須**像她會放在自己 IG 的東西**，不能像系統推播。

### 2.5 黃品璇（接案+微型主 41 女）─ 成熟從容

**她想成為**：Kinfolk magazine 那種生活美學家 / 工作室主理人 / 已經「到那個位子」的設計師 ── 不再為錢焦慮、可以選擇接什麼案、家庭與工作平衡。

**視覺想像**：
- 場域：日光從側邊進來的書桌、一杯茶、紙本筆記
- 工具參照：**Kinfolk / Cereal Magazine / 日本書店 / 蔦屋書店** ── 安靜的成熟
- 字體：明朝體 / 思源宋體 / 中文襯線是核心 ── 她讀中文的比例最高
- 色彩：自然白、深棕、橄欖、不是高彩度
- 排版：文字為主、圖小、留白多
- 厭惡：CTA 強壓、「立即報名」「限時優惠」、訊息密度高

**她成功時的列印**：她把月度 mentor 信印出來貼在書桌前。**設計含義**：Email 模板必須**列印 A4 後仍美觀**，不依賴螢幕色彩。

## 3. 5 個想像的交集 ─ 視覺真理

5 個 persona 的成功想像看似分散（開發者 / 作者 / 顧問 / 品牌主 / 設計師），但**他們對「成功的工具長什麼樣」的交集極清楚**：

### 3.1 共通的 7 個視覺真理

| # | 真理 | 為什麼 5 個 persona 都同意 |
|---|---|---|
| 1 | **大量留白** | 留白 = 我不需要每個位子都填滿東西 = 我有餘裕 = 我已經到了 |
| 2 | **單一強調色** | 多色 = 不確定 = 想討好所有人 ─ 成熟者敢只用一個顏色 |
| 3 | **字重對比強** | 標題與內文字重對比鮮明 = 訊息有層級 = 我尊重你的閱讀時間 |
| 4 | **不油的字** | 沒有「✨」「🚀」「!!」「立即」── 這些都是「想急著賣你」 |
| 5 | **不用 stock photo** | Stock photo 笑臉女生 = 沒個性 = 我可以是任何 SaaS |
| 6 | **截圖好看** | 5 個 persona 都會截圖（公開 / 私下 / 私藏）── UI 在脫離脈絡時仍要美 |
| 7 | **長期不過時** | 不用 2024-2026 流行的漸層 / 玻璃擬態 / neumorphism ── 5 年後還能看 |

### 3.2 共通的 5 個情緒目標

當訪客滑商品頁、客戶用 LINE Bot、用戶讀 email 時，期待的情緒體感：

1. **「他們懂我在意什麼」** ─ 用詞精準、不假掰、不被當提款機
2. **「這是給認真的人用的」** ─ 不像「人人都該訂閱」的 broad market 工具
3. **「我可以放在自己 IG / Twitter / 書桌而不丟臉」** ─ 美感外溢
4. **「這不是急著賣我」** ─ 沒有焦慮 funnel、沒有 chat widget 跳出
5. **「我已經是這個圈子的人了」** ─ 訂閱前是嚮往，訂閱後是身份

## 4. 風格主軸（給 design 的 brief）

### 4.1 整體調性 (Mood)

**一句話**：**「東京 / 京都的書店 + 矽谷的開發者工具 + Substack 的寫作者 ── 三個交集」**。

不是混搭。是這三者本來就有共同基因：
- **節制** ─ 該說的說，不說的留白
- **權威** ─ 不假掰，知道自己在做什麼
- **可被信任** ─ 看起來會在 5 年後還在這裡

**Mood Board 收集方向**（給 claude design 起手）：
- 蔦屋書店、誠品敦南（已歇業，但精神）室內攝影
- Stripe / Linear / Vercel / Raycast 的官網
- Substack 個人作者頁面（找有設計感的，不是預設模板）
- Aesop / Glossier / Sun Day 品牌官網
- Kinfolk / Cereal Magazine 排版
- a16z / Stratechery 內容版面

**避免參照**：中文知識付費平台、SaaS hero illustration libraries、Webflow templates 中「business」分類、Dribbble 首頁的 trending。

### 4.2 色彩語意（不給 hex、給語意）

claude design 需挑出具體 hex，但約束如下：

#### 主背景

- **語意**：紙感、不純白、有一點溫度
- **方向**：象牙白 / 米白 / 微暖灰白
- **反例**：純 #FFFFFF（顯廉價）、深色背景（不是這個產品的調性，dark mode 留 v1.5 客戶後台）

#### 主文字色

- **語意**：墨色、不純黑、長時間閱讀不刺眼
- **方向**：深墨灰、深棕黑
- **反例**：純 #000000、亮藍灰（顯 corporate）

#### 強調色（單一）

- **語意**：「主理人選的那一個顏色」── 像作家挑墨水
- **方向（候選）**：
  - 深藏青（成熟知識感）
  - 焦糖 / 老金（不是金光，是書封燙金後氧化的感覺）
  - 墨綠（茶 / 苔 / 圖書館）
- **使用紀律**：強調色 **只用在 CTA、引用線、mentor accent**。內文連結用底線而非變色，留強調色的稀缺感
- **反例**：紫色漸層、橘紅、霓虹綠、品牌色用很多次（用了就不強調）

#### Mentor 4 色（次要層）

每位 mentor 對應一個次要強調色，僅在「Mentor 介紹卡」「Mentor 視角的 email accent line」「客戶後台 mentor 標記」用：

- **Jobs**：純黑（產品本質、品味）
- **Buffett**：深焦糖 / 老金（時間感、長期）
- **Naval**：墨綠 / 苔（自然、永久遊戲）
- **Musk**：鋼灰 / 深鈦（工程感、第一性原理）

claude design 拍板具體值時注意：4 色彼此色相距離夠、彩度都低、不會在同一頁同時亮起來。

### 4.3 字體方向

#### 中文（主）

- **內文 / 長文**：思源宋體 (Source Han Serif) / 思源黑體 (Source Han Sans) 二選一 ── 取決於該頁性質
  - 商品頁長段落、月度 mentor 信、retro pre-brief email：**宋體**（黃品璇、林思婕的閱讀偏好）
  - 客戶後台、表單、status 顯示：**黑體**（陳定遠、吳沛文、張凱恩偏好）
- **標題**：思源宋體 加重、或專業中文字型如 **王漢宗、justfont 系列、文鼎 UD 晶熙**（claude design 挑）
- **絕對不用**：標楷體、華康少女體、超粗黑、毛筆風

#### 英文（副，但精準很重要）

- **內文**：geometric sans 或 humanist sans（如 Inter / Söhne / Mona Sans 路線）
- **標題**：可選 serif（如 Tiempos / Source Serif / GT Sectra 路線）襯托知識感
- **等寬**（用於程式 / 數字 / 時間戳）：JetBrains Mono / IBM Plex Mono 路線

#### 中英混排紀律

- 中英文本字間距 letter-spacing 微調（中文略寬、英文預設）
- **標點不用全形與半形混搭**：中文段落純全形、英文段落純半形
- 數字（如 NT$ 599、12 sprints）用 **tabular nums** 確保對齊

### 4.4 構圖 / 排版

#### Hero（首屏）

- **不超過 3 句話**（主標 + 副標 + 一句 sub-hook）
- **大量留白**：首屏文字佔比建議 < 40%
- **CTA 一個**（「7 天免費試用」），不要左 CTA + 右 CTA + 底部 CTA 三連發
- 不用首屏輪播 / 自動切換動畫
- Demo 影片**不自動播放有聲**

#### 區塊節奏

- 區塊間距 **比常見 SaaS 大 1.5 倍**（讓「重量感」出來）
- 區塊內容居中對齊**不是預設**：依 persona 偏好，文字內容區塊**左對齊**為主（如 Substack / 寫作者頁面），居中只在 Hero 主標、price card 等明確需要

#### Grid

- 12-col grid 預設，但內容**最大寬度受限**（max-width，類似 Substack 的 ~720px 內文）
- **不滿版**（不要全寬 hero / 全寬照片淹沒視覺）

#### 視覺層級

- **3 層字級對比強**：H1 / H2 / Body 體感差異要明顯（粗細 + 大小 + 行距三維度差）
- 強調色 / 次要色 / 線條使用要克制：**一頁不超過 5 個視覺強調點**

### 4.5 動效

**整體原則**：**動效必須「服務閱讀」，不是「服務炫技」**。

- ✅ 滾動觸發的 fade-in（短、不誇張）
- ✅ hover 微妙的 underline / 顏色加深
- ✅ 載入時的骨架屏（skeleton）替代轉圈圈
- ❌ 視差捲動（parallax）── 顯廉價
- ❌ 文字逐字打出 ── 浪費讀者時間
- ❌ 大區塊的 scale / rotate ── 顯新創 vibe
- ❌ Lottie 帶笑臉的 illustration 動畫 ── 與整體調性衝突
- ❌ 自動跑的 hero 元素

### 4.6 影像 / 插畫 / 圖示

#### 攝影

- **不用 stock photo**（5 個 persona 都會嫌）
- 必要時用：
  - 主理人 Fred 的真實照片（書桌、工作環境）── 強化「人感」
  - 產品截圖（LINE 對話、後台介面、email 範本）── 截圖品質決定一切
  - **自然元素**（紙、墨、書頁、植物、光影）── 可少量點綴

#### 插畫

- **MVP 階段不用插畫**（風險過高，做不好就破壞調性）
- 若 v1.5 後要用，方向：**極簡線條 + 單色 + 不擬人**（類似 Stripe docs / Linear 用的圖）

#### 圖示

- **線條圖示**（outline icon）為主，不用填色
- **不用 emoji 作為功能圖示**（emoji 在不同系統渲染不一致、顯非正式）
- 常用工具：Lucide / Heroicons / Phosphor 路線（claude design 挑一套）

### 4.7 文案口吻

> spec §2.3 已定調「直接、行動導向、不柔化術語」。本文件補上**設計含義**：

- **不用感嘆號**（「立即試用！」「快來體驗！」）
- **不用「我們」開頭的客套**（「我們相信...」「我們致力於...」）
- **不用「您」**（用「你」更直接）
- **不用「親愛的朋友」「夥伴們」**
- **數字直接寫**（「2 天一個 sprint」「NT$ 599」）── 不用「短短兩天」「不到六百元」
- **每句話有觀點**：不寫「成功需要努力」這種正確但無內容的句子

## 5. 反例（一定不要）

把這份清單貼在 claude design 顯示器旁。這些是 5 個 persona **共同會立刻關掉頁面** 的元素：

### 5.1 視覺反例

- ❌ 紫粉漸層 hero
- ❌ Illustration 帶笑臉的卡通人物
- ❌ Stock photo（笑臉女生 / 握手 / 多元團隊照）
- ❌ Glassmorphism / Neumorphism（過時 + 廉價感）
- ❌ 大面積純黑底 + 螢光色字（電競 / web3 vibe）
- ❌ 倒數計時器、「限時優惠」紅色 banner
- ❌ Chat widget 浮動按鈕
- ❌ Cookie banner 全頁 modal
- ❌ Newsletter 訂閱 popup（尤其是「等等！再看一眼！」exit intent）
- ❌ 多個 CTA 按鈕在同一屏
- ❌ 「Trusted by」logo 牆（5 個 persona 都不認 logo、且我們 MVP 沒 logo 可放）
- ❌ 證言區大頭照 + 三星五星評分

### 5.2 文案反例

- ❌ 「Empower your business」「Unlock your potential」「Transform your life」
- ❌ 「成功不是偶然」「改變從這一刻開始」「你值得擁有」
- ❌ 「親愛的朋友」「我們相信」「讓我們一起」
- ❌ 連續兩個感嘆號 / 三個刪節號
- ❌ 「業界第一」「最好的」「唯一的」（顯不誠實）
- ❌ Emoji 開頭的標題（🚀 立即啟動）
- ❌ 「為什麼選擇我們？」式自問自答區塊

### 5.3 互動反例

- ❌ 強制 email 註冊才能看 pricing
- ❌ Pricing 用「Contact us」隱藏
- ❌ 試用要先填完整公司資訊
- ❌ 「再看 30 秒就走」的退出彈窗
- ❌ Live chat 主動跳出問「需要幫忙嗎」
- ❌ 滾動到底自動觸發訂閱表單

## 6. 各區塊風格體感（依 spec §8 對應）

每個區塊的「設計成功」標準。claude design 拿這份做設計驗收清單。

### 6.1 商品頁 Hero（spec §8.1）

**體感目標**：訪客看 3 秒就知道「這不是常見的 SaaS」。

- 主標居中，三句話以內
- 一個 CTA、底色微妙（不是亮色填充）
- Demo 影片下方而非右側 ── 強調主標不是配角
- 背景無 illustration、無漸層
- **訪客滑進來時感覺**：「這是某個認真做事的人開的網站」

### 6.2 Mentor 介紹區（spec §8.1 / §11.4）

**體感目標**：4 位 mentor 像「圖書館的 4 個書架」，每位有自己的視覺氣質。

- 4 張卡片並排（桌機）/ 上下堆疊（手機）
- 每張卡片：
  - 中英並列名（ spec §V2 「賈伯斯 (Steve Jobs)」格式）
  - 一句定位
  - 適用議題 3 條
  - 經典提問 2 條（letterspacing 略寬，仿引用感）
  - **真實性聲明可摺疊**（預設展開，可收合，避免每張卡都一塊長文）
- 每位 mentor 對應 §4.2 的次要色，作為**一條細線 / 一個小點**，不是大面積色塊
- 不放 mentor 真人照片（侵權風險 + 真實性聲明衝突），用幾何符號 / 抽象插畫 / 中文名印章感（claude design 挑）

### 6.3 Pricing（spec §3.1）

**體感目標**：「乾淨、有底氣」，不像在賣課。

- 單一卡片（MVP 只有一個方案）
- NT$ 599 / 月、7 天免費試用、首月 7 天退款 ── 三點直接列
- 沒有「最受歡迎」標籤（沒得比）
- 沒有「原價 NT$ 1,200」劃掉那種營造優惠感
- CTA「7 天免費試用」與 Hero CTA 文案完全一致

### 6.4 FAQ（spec §8.1）

**體感目標**：像「讀者在書店翻一本書的目錄」，不是「銷售頁的反對處理」。

- 預設展開**前 3 題**（最常被問的：跟 ChatGPT 差別、試用退款、Mentor 真實性）
- 其他可展開
- 答案排版**像短文**，不是 bullet 點滿
- 連結（如「隱私政策」）用**底線**而非藍色
- 每題答案 < 80 字，超過就拆段

### 6.5 LINE Welcome Flex Message（spec §8.3.2）

**體感目標**：吳沛文截圖發 IG 不丟臉、林思婕收藏不嫌俗。

- 卡片色：與網站主背景同（象牙白），不用 LINE 預設深色
- 文案 4 行內：定位 + 4 mentor 縮圖列 + 一句行動 + 真實性聲明縮寫
- Mentor 縮圖**不用真人**，用抽象符號 + 中英名
- 行動句：「準備好就傳訊息給我。」── 不是「立即開始你的 sprint！」

### 6.6 Onboarding 7 步問卷（spec §3.3）

**體感目標**：「填的時候不覺得被審問，覺得被認真對待」。

- 一頁一題（不要長表單一次塞 7 題）
- 進度條淡，不放最上方 ── 不要「還有 6 題」的壓迫感
- 每題上方一段引導文字（2-3 行），解釋**為什麼問這題**
- Step 4 / Step 2 對應 [customer-journeys.md](customer-journeys.md) §9 衍生需求：
  - **Money Model 提供 4 範本**（電商 / 服務業 / 內容創作 / 諮詢）
  - **想推進的事改為「主任務 + 最多 2 個支線」**
- 每步「下一步」按鈕不用強調色 ── 不是急著推進
- 「上一步」與「下一步」同等視覺權重

### 6.7 客戶後台（spec §8.4）

**體感目標**：「我每天打開都不討厭」── 黃品璇 / 林思婕的高頻使用情境。

- 側欄極簡：5 個一級項目（對話 / 記憶 / 風格 / 帳單 / 資料）
- **記憶面板**是後台主角（spec §8.4.2 v1 必上線）：
  - 卡片式陳列，每張卡片像「索引卡」
  - 編輯狀態極輕，不用 modal
  - 可加標籤、可排序
- 對話歷程**像翻日記**，不像「客服紀錄」
- 訂閱管理區避免「降級 / 取消」按鈕做得像紅色危險區 ── 用同等視覺層級，不嚇唬

### 6.8 Email 模板（Resend / spec §10.4）

**體感目標**：「印出來貼書桌不丟臉」（黃品璇）+「截圖發 IG 不掉價」（林思婕、吳沛文）。

#### 共通骨架

- 寄件人名：「Next 夥伴 / {Mentor 中文名}」── 對應 mentor 視角
- 主旨：**不用 emoji**、不用「!」、不用「【】」括號
- 標題下方一條細線（強調色），是視覺記憶錨點
- 內文寬度受限（~ 600px），左右大留白
- Footer：簡短版權 + 一行退訂連結 + 真實性聲明縮寫
- **不放品牌彩色 logo / 大圖 banner**

#### 動態 mentor 信（pre-brief / final-call / 月度信）

- mentor 中英名作為標題 prefix
- 該 mentor 次要色作為**引用線**（左側細直線）
- 內文像書信（不像 marketing email）
- **整封信印出 A4 後仍美觀**（黃品璇驗收標準）

## 7. 跨 Persona 的差異層處理

5 個 persona 對主軸有共識，但在**閱讀字體偏好**、**資訊密度**、**emoji 接受度**有差異。設計策略：

### 7.1 主軸不變、差異放在客戶後台「個人化」設定

- 客戶後台 `/account/persona` 加一個「閱讀風格」選項：
  - 「文（宋體 / 寬鬆）」── 林思婕、黃品璇預設
  - 「工（黑體 / 緊湊）」── 陳定遠、張凱恩預設
  - 「品（混排 / 留白）」── 吳沛文預設
- 影響範圍：對話歷程顯示、記憶面板字體
- **商品頁與 onboarding 不分支**（避免初次體驗的 A/B 焦慮）

### 7.2 Email 模板內依 mentor 視角差異化（不依 persona）

- mentor 視角已經承載差異：
  - Jobs 視角：極簡、銳利
  - Buffett 視角：書信感、長段落
  - Naval 視角：哲學、引用感
  - Musk 視角：條列、緊湊
- claude design 為 4 個 mentor 視角設計**子模板**，主模板不變

## 9. 給 claude design 的開工清單

### 9.1 第一階段任務（建議順序）

1. **Mood board 建立**：依 §4.1 收集 30-50 張參照圖，標註「為什麼這張」
2. **色彩三選一**：從 §4.2 強調色候選（深藏青 / 焦糖 / 墨綠）做 3 個版本給 Fred 選
3. **字體配對提案**：中英文字體 + 等寬字 ── 各做 2-3 套配對
4. **Hero 提案 3 版**：用最終色彩 + 字體做商品頁 Hero 三個版本，差異點：留白程度 / 字級 / Demo 影片位置
5. **Mentor 卡片提案**：4 位 mentor 的視覺呈現（避免真人照片的解法）
6. **Email 主模板**：Resend React Email 主框架

### 9.2 開工前必讀

- [success-agent-service-spec.md](success-agent-service-spec.md) §1.4 真實性聲明、§8 功能規格
- [customer-personas.md](customer-personas.md) §2-6 五位 persona 全文
- [customer-journeys.md](customer-journeys.md) §3.6（陳定遠 LINE Welcome）、§4.7（林思婕 Sprint #1）、§7.7（黃品璇 Sprint #1）── 三段是體感最關鍵的旅程描述
- 本文件 §5 反例 ── 貼在顯示器旁

### 9.3 設計卡關時的判斷句

當任何設計決策卡住，回到三個問題：

1. **「這個元素，5 年後看起來還會 OK 嗎？」**（時間感）
2. **「林思婕 / 吳沛文截圖到 IG，會不會丟臉？」**（外溢美感）
3. **「陳定遠 / 張凱恩看到，會不會覺得這是給小孩玩的？」**（被尊重感）

三個都通過，才算過關。
