# Next 夥伴 ｜ 客戶圖像 (Customer Personas)

## 0. 文件定位

本文件依據 [success-agent-service-spec.md](success-agent-service-spec.md) §2「目標客戶」與 §1.3「三個關鍵差異化」，並交叉比對 [csm-agent-plan-v2.md](csm-agent-plan-v2.md) §1.2「Mentor 庫定位」、[service-insight-initial.md](service-insight-initial.md) 中「一人公司 7 系統 / Dickie Bush 8 文件」框架，建立 **5 個具備可操作細節的客戶圖像**。

**用法**：
- 行銷文案、商品頁見證、Onboarding 問卷措辭，皆以這 5 個圖像為投射對象
- 每位 persona 對應 [customer-journeys.md](customer-journeys.md) 中一條完整旅程
- Phase 0 種子用戶招募名單（spec §15 開放議題）以本文件五型為配額參考

**範圍外**（本文件不展開，留給其他文件）：
- 行銷渠道與媒介選擇（屬 Phase 2 行銷投放規劃）
- 個別 mentor 的 prompt 設計細節（屬 `agents/mentors/` 內容）
- Onboarding 問卷的具體問句文案（屬 spec §15 待開放議題）

## 1. Persona 設計方法

### 1.1 採樣維度

依 spec §2.1 客群側寫表的 5 個維度展開，並補上 **產品節奏對位**（mentor 偏好、sprint 週期、推播時間）3 個產品內部維度：

| 維度 | 採樣範圍 |
|---|---|
| 角色類型 | SaaS Indie / 知識變現 / 副業上班族 / D2C 創業 / 自由工作者+微型主 |
| 年齡 | 27 ─ 45 歲 |
| 性別 | 男 2、女 3（對應台灣自由工作者 / 個人品牌經營者性別比） |
| 地理 | 台北 2、新竹 1、台中 1、高雄 1（涵蓋四大都會區） |
| 月可支配工具預算 | NT$ 1,500 ─ NT$ 8,000 |
| 主導痛點 | 對應 7 系統中各自最弱項（內容 / 產品 / 銷售 / 交付 / 客戶 / 行政 / 復盤） |
| 預期 mentor 命中率 | 4 位 mentor 各至少 2 個 persona 對應 |
| Sprint 結束時間錨點 | 4 個錨點（12:00 / 17:00 / 21:00 / 23:59）皆有 persona 偏好 |

### 1.2 排除哪些不畫

依 spec §2.2「不在客群」原則，**不為下列群體建立 persona**，避免行銷誤投：

- 純情緒陪伴 / 心理諮商需求者
- 18 歲以下
- 完全抗拒「sprint」「截止時間」「被推進」概念者
- 期待免費或低於 NT$ 199/月者
- 期待「找代寫 / 找代執行」的客戶（Next 夥伴是推進，不是代工）

### 1.3 真實性紀律

每個 persona 都附上 **明確可被驗證 / 反證的事實段**（職稱、薪資範圍、使用過的工具品牌、訂閱過的內容），方便日後用真實訪談資料校準，避免 persona 變成虛構神話。種子用戶實際訪談後，本文件 v2 會列出哪些假設被打中、哪些被推翻。

## 2. Persona 1 ─ 陳定遠｜SaaS Indie Hacker

> **一句定位**：「我已經 launch 過兩個收掉的 side project，這次想找個會盯我的同事。」

### 2.1 基本資料

| 維度 | 內容 |
|---|---|
| 年齡 / 性別 | 32 歲 / 男 |
| 地點 | 台北市內湖區 |
| 職業 | 外商科技公司資深軟體工程師（Backend / Platform） |
| 家庭 | 已婚、無小孩、與妻共住 |
| 年薪 | NT$ 220 萬（含 RSU） |
| 教育 | 國立大學資工碩士 |
| 工作模式 | 正職 9-19 時、side project 晚上 21-24 時 + 週末早上 |

### 2.2 與本產品相關的事實

- **付費工具現況**：ChatGPT Plus（USD 20）、GitHub Copilot（USD 20）、Linear 個人版、Notion Plus、Vercel Pro、Cursor Pro。月固定 USD 100+ 工具支出，已習慣 SaaS 訂閱
- **過去兩年 Side Project 紀錄**：
  - Project A：開發者工具，2024 launch，3 週後停更（沒有第一個付費客戶）
  - Project B：B2B SaaS for marketing team，2025 launch，2 個月後決定收掉（PMF 不對）
- **目前進行中**：Project C（無代碼自動化工具的開發者版），已寫 60% 程式碼但 6 週沒進展
- **常逛內容源**：Indie Hackers、Y Combinator Startup School、Hacker News、Pieter Levels Twitter、Naval podcast 全集、Lenny's Newsletter
- **熟悉術語**：sprint、PMF、ARR、MRR、TAM、burn rate、launch、ship、retention

### 2.3 一天的樣子（節錄）

> 「9 點到公司站會，照常推主線。午餐邊吃邊滑 Twitter 看其他 Indie 有沒有 launch。下班 7 點到家，陪老婆吃飯到 9 點。9-12 點理論上是 side project 時段，但有一半的天會變成『我研究新工具』『我重構檔案結構』『我看 YouTube 別人怎麼做』，真正寫產品 code 的時間每週可能不到 6 小時。週五晚上常告訴自己『週末好好衝』，週六早上起來其實只有 1 小時專注，下午就被生活其他事吃掉。」

### 2.4 心理側寫

| 面向 | 內容 |
|---|---|
| **長期目標** | 3 年內讓某個 SaaS 達到 USD 5k MRR，能不能離職另說，至少證明自己能做出有人付費的東西 |
| **本季目標** | Project C 在 6 週內 launch，3 個月內 first paying customer |
| **挫折感** | 「我技術沒問題，產品也想得清楚，但每次都死在最後一哩。一個人寫沒人盯，會無限期延後 launch 日期」 |
| **動機根源** | 不想 50 歲還是「會 ship 但沒 ship 過自己東西」的工程師；想證明 craftsman 可以是 entrepreneur |
| **恐懼** | 又是一個半年後收掉的 project；老婆開始不耐煩；同期同事都已經有 side income |
| **對被人推進的態度** | 完全可以接受，甚至渴望。曾考慮過花 USD 200/月找 accountability coach 但覺得太貴 |

### 2.5 對 Next 夥伴的觸發點與抗拒點

**觸發點（為什麼會想試）**：
1. 「2 天一個 sprint」直接命中他「side project 沒節奏」的痛點，對 sprint 概念也已經熟悉
2. Mentor 中有 Naval 與 Musk，是他長期追的兩位，「他們會怎麼看我這個 sprint」具誘惑力
3. NT$ 599/月 = USD 19，比一個 ChatGPT Plus 便宜，比 accountability coach 便宜 10 倍
4. 7 天免費試用 + 首月退款，下決定的成本極低

**抗拒點（為什麼可能不付費）**：
1. 已經訂太多工具，每多一個都要說服自己「這個會用」
2. 對中文 AI 產品有疑慮，預期會問「能不能用英文跑？」
3. 擔心 Mentor 框架「太抽象」、不如直接問 Claude 或 ChatGPT 來得實用
4. 妻子可能會問「你又訂一個」

### 2.6 預期使用模式

| 維度 | 預測 |
|---|---|
| 偏好 mentor | **Musk 主軸**（執行加速、砍範圍）+ **Naval 副軸**（槓桿選擇、長期遊戲） |
| Sprint 週期 | 2 天（默認） |
| Sprint 結束時間 | 23:59（deadline-driven，符合工程師壓底習慣） |
| 對話風格 | cold（簡潔、不油） |
| 推播管道 | LINE push + Email 都開（不想錯過 final-call） |
| 主要使用 skill | sprint-plan / sprint-checkin / launch-gate |

### 2.7 願付價格與留存風險

| 維度 | 評估 |
|---|---|
| **試用 → 付費機率** | **高（80%+）**：價格不是門檻，只要 Sprint #1 跑通就會留 |
| **預期 LTV** | 12 個月以上（會用到 Project C launch 後再評估） |
| **流失最高風險** | Project C 在第 4-6 週確定要 pivot 或收掉 → 失去使用情境 |
| **流失緩衝** | 這群人通常會立刻啟動 Project D，本產品的記憶面板讓 Agent 能無痛接續 |
| **可升級空間** | v1 後若有「Team / Premium 自訂 mentor」，會升級（曾為 Lenny's Newsletter 升 Pro） |

### 2.8 對應 spec 段落

- spec §2.1「角色：SaaS 獨立開發者」直接命中
- spec §1.3 差異化 #1（Mentor 思想框架）的最佳 fit
- spec §3.3 高度客製導入的「沉默成本」對他無壓力（願意花 10 分鐘填問卷）
- spec §6.1 LINE 通道：使用率高，但**不是 LINE 重度使用者**，可能更偏好 Email push

## 3. Persona 2 ─ 林思婕｜知識變現創作者

> **一句定位**：「Substack 5,000 訂閱、IG 18,000 追蹤，但這個月又只發了 2 篇。我需要的不是更多靈感，是一個逼我交稿的人。」

### 3.1 基本資料

| 維度 | 內容 |
|---|---|
| 年齡 / 性別 | 36 歲 / 女 |
| 地點 | 台中市西屯區 |
| 職業 | 全職個人品牌經營者（職涯教練 + 寫作者），3 年前從外商人資轉型 |
| 家庭 | 已婚、無小孩、養 1 隻貓 |
| 年收入 | NT$ 110 萬 ─ NT$ 160 萬（變動大） |
| 教育 | 國立大學企管系、英國碩士 |
| 工作模式 | 自由排程，但實際上「早上拖延、下午有點動、晚上才衝刺」 |

### 3.2 與本產品相關的事實

- **內容矩陣**：
  - Substack 雙週電子報（付費 USD 5/月，目前 5,200 訂閱中 320 付費）
  - Instagram 18,000 追蹤
  - Threads 4,000 追蹤
  - 一年 4 場線下工作坊（每場 30 人 NT$ 3,500）
  - 一對一職涯諮詢（NT$ 4,800/小時，每月 8-12 小時）
- **付費工具現況**：Notion Plus、Substack Pro、Canva Pro、Calendly、Riverside、ChatGPT Plus、Granola
- **訂閱過的「成長系」內容**：Lenny's Newsletter、Justin Welsh、Dickie Bush 寫作課、職人簡報書
- **熟悉術語**：sprint（從敏捷小組讀過）、OKR（前公司用過）、launch（懂、剛 launch 過第一個線上課）、streak（James Clear 帶起來的）、 first principles

### 3.3 一天的樣子（節錄）

> 「我可以排得很滿，也可以一整天什麼都沒做。早上常從 IG 滑到 11 點才坐下來打開 Notion，看到 5 個未完成的 draft 就先處理 email、回客戶、改下次工作坊的 PPT。下午有客戶就還好，沒客戶就『今天好像沒交付出去什麼』。晚上 9 點以後反而最有產能，但這種『只有夜晚有效』的工作模式我已經受夠了。我想要的是『有人在禮拜二中午跟我說：你昨天說要寫的那篇還沒交』。」

### 3.4 心理側寫

| 面向 | 內容 |
|---|---|
| **長期目標** | 3 年內把線上課做成 NT$ 200 萬/年的穩定收入，從「靠時間賣諮詢」轉成「資產型」 |
| **本季目標** | 線上課第二期招生（5 月底），同時把 Substack 從雙週轉週更 |
| **挫折感** | 「我有靈感、有材料、有讀者，但每週都會有 1 ─ 2 天什麼都沒生出來。寫作教練、生產力工具、自我管理書都看過，但真實情況是還是會放空整天」 |
| **動機根源** | 害怕「個人品牌做著做著就消失」；同期創作者已經有人月入 30 萬，自己 8 萬 ─ 12 萬上下波動 |
| **恐懼** | 訂閱者開始流失、商業模式回不去找全職工作、「自由工作的代價是不自由」 |
| **對被人推進的態度** | 表面歡迎、實際抗拒。需要的是「溫柔但不可妥協」的 mentor，不是嚴厲教官 |

### 3.5 對 Next 夥伴的觸發點與抗拒點

**觸發點**：
1. Jobs（產品本質、品味）對「我這篇值不值得發出去」這個自我質疑直接有用
2. Buffett（長期、護城河）對「我的個人品牌護城河是什麼」這個焦慮提供結構
3. 訂閱費 NT$ 599 = 一杯 Blue Bottle + 一個甜點，心理門檻低
4. **記憶透明面板**對她「擔心 AI 不懂我」的疑慮直接解套
5. LINE Bot 對她「中午不想開電腦」的工作節奏完美契合

**抗拒點**：
1. 對「sprint」一詞有點 corporate 感，會想知道能不能改成「一檔」「一輪」
2. 會擔心「2 天太短，創作不是這樣的」── 這是 onboard 階段需要說服的關鍵
3. Mentor 4 位都是男性西方科技 / 投資人，可能反問「沒有女性 mentor 嗎」
4. 不想被 cold 對話風格刺到，預設選 passionate

### 3.6 預期使用模式

| 維度 | 預測 |
|---|---|
| 偏好 mentor | **Jobs 主軸**（內容品質、刪減）+ **Buffett 副軸**（長期建構） |
| Sprint 週期 | **嘗試 2 天，但會在第 2-3 個 sprint 改成 1 週**（要 onboard 階段預判處理） |
| Sprint 結束時間 | 21:00（晚間族，預設值） |
| 對話風格 | passionate（預設） |
| 推播管道 | Email 主、LINE push 開但不依賴 |
| 主要使用 skill | sprint-plan / retro / profile-deepen（最常聊「我發現自己其實...」） |

### 3.7 願付價格與留存風險

| 維度 | 評估 |
|---|---|
| **試用 → 付費機率** | **中高（65%）**：取決於 Sprint #1 retro 的觸動感 |
| **預期 LTV** | 18 個月以上（個人品牌經營是長線），但中間可能停 1 個月「休息」 |
| **流失最高風險** | Sprint 第 1-2 週若覺得 mentor 視角「太硬」，會直接退訂 |
| **流失緩衝** | 月度 mentor 信（spec §7.3）對她特別有黏性，會被截圖轉發 IG |
| **推薦動機** | **極高**：個人品牌經營者本身就是 KOL，會在 Substack / IG 寫使用心得（潛在獲客槓桿） |

### 3.8 對應 spec 段落

- spec §2.1「角色：副業推進中的上班族」延伸到「全職個人品牌經營者」這個未明列但同質的子群
- spec §1.3 差異化 #3（記憶透明）對她的「AI 焦慮」是核心吸引力
- spec §8.4.2 記憶面板上線必須是 v1（不能延），對留住這型客戶極關鍵
- spec §11.4 mentor 真實性聲明：她對「不是真的 Jobs」這件事接受度高，但會放大檢視措辭

## 4. Persona 3 ─ 張凱恩｜副業驗證上班族

> **一句定位**：「我有正職、有家庭、有副業、沒時間。週末 4 小時的副業時間我必須花在對的事情上，不能再亂試。」

### 4.1 基本資料

| 維度 | 內容 |
|---|---|
| 年齡 / 性別 | 38 歲 / 男 |
| 地點 | 新竹市東區 |
| 職業 | 半導體公司軟體技術主管 |
| 家庭 | 已婚、2 個小孩（5 歲、2 歲）、與配偶雙薪 |
| 年薪 | NT$ 280 萬 |
| 教育 | 國立大學電機博士肄業（轉業界） |
| 工作模式 | 正職 8-19 時 + 通勤 1.5h；副業每週可分配 8-10h（晚上 22-24 時、週六上午） |

### 4.2 與本產品相關的事實

- **副業構想**：B2B 半導體業 AI 導入諮詢顧問，目標 3 年內離職全職做
- **副業現況**：寫了 12 篇 LinkedIn 長文（追蹤 800+），談了 2 次免費諮詢，但沒收到第一筆錢
- **付費工具現況**：ChatGPT Plus、Notion 個人版、LinkedIn Premium、Audible
- **訂閱過的「諮詢顧問起步」內容**：Justin Welsh、Pat Walls 的 Starter Story、Daniel Priestley 的 Key Person of Influence 書
- **熟悉術語**：sprint（公司用 SAFe）、OKR（前公司）、launch、PMF（讀過 lean startup）

### 4.3 一天的樣子（節錄）

> 「平日下班 7:30 到家，陪小孩到 9:30，太太把小孩弄睡後我們會聊 30 分鐘，10 點才是『我的時間』。但常常一坐下就累，刷 LinkedIn、看完一集 Podcast 就 12 點。週六早上太太帶小孩去娘家是我最寶貴的 4 小時，但這 4 小時的 ROI 我從來沒算清楚 ── 上週寫一篇 LinkedIn 文，這週設定一個諮詢 funnel，下週讀競品，三週下來『副業有進展』但沒有任何具體成果。」

### 4.4 心理側寫

| 面向 | 內容 |
|---|---|
| **長期目標** | 3 年內副業月入 NT$ 15 萬以上，太太也願意他離職時就辭 |
| **本季目標** | 收到第一筆付費諮詢（任何金額）、把 LinkedIn 追蹤推到 2,000 |
| **挫折感** | 「時間真的有限。我不需要『更多想法』，我需要『這 4 小時就做這件事』的明確指令」 |
| **動機根源** | 不想 45 歲還在科技業當主管被裁；想證明自己離開大公司也能養家 |
| **恐懼** | 副業驗證失敗，但時間已經過了 2 年；錯過離職黃金窗口 |
| **對被人推進的態度** | 公司本來就在推進他，副業也想要被推進（已內化 sprint 文化） |

### 4.5 對 Next 夥伴的觸發點與抗拒點

**觸發點**：
1. Buffett「10 年後這個還在嗎」對他「我的諮詢能不能變成 IP」直接命中
2. Naval「永久遊戲」對「離職創業」的長線思考有用
3. 7 天試用無成本，可以週六試一次 sprint
4. NT$ 599 月薪 1/4500，完全不是預算問題（公司每月給他 NT$ 3,000 學習津貼可報帳）
5. LINE Bot 在通勤時可以對話，符合他「碎片時間用」習慣

**抗拒點**：
1. 「2 天 sprint」對他副業節奏太快，需要被勸到「1 週 sprint」
2. 擔心 onboarding 7 步問卷在週六早上拖掉太多時間
3. 不確定 AI 能不能理解半導體業的特殊脈絡（B2B 利基）
4. 工作隱私顧慮：「公司資料不能寫進去吧？」── FAQ 必須回應

### 4.6 預期使用模式

| 維度 | 預測 |
|---|---|

## 8. 衍生設計建議（給後續文件）

依據本 5 個 persona 的交叉觀察，以下建議**僅列項目**，不在本文件展開，留給對應的後續文件：

### 給商品頁文案（spec §8.1）

- Hero 主標除「2 天一個 sprint」外，再準備一條「給 1 週 sprint 也行」的 sub-hook（吸納張凱恩、黃品璇）
- Mentor 卡片要明確露出「適用議題」── 林思婕、吳沛文都會用「他們懂我這行嗎」來判斷
- FAQ 必加：「如果我不熟 sprint 概念怎麼辦」、「公司資料寫進去安全嗎」（張凱恩）、「中文回應品質」（吳沛文）

### 給 Onboarding 7 步問卷（spec §3.3 / §15）

- Step 4 Money Model、Step 5 Avatar Map 對黃品璇型最摩擦 → 提供「接案 / 教學型」替代問句版本
- Step 6 sprint 偏好預設 2 天，但 UI 上要明確露出「1 週 / 自訂」選項，避免林思婕、張凱恩在這一步離開
- Step 7 推播 email 要強調「不會用於行銷」，緩解張凱恩的「公司隱私」顧慮

### 給 LINE Welcome Flex Message（spec §8.3.2）

- 4 位 mentor 縮圖下方加一句「指派由 AI 視議題自動」，避免吳沛文以為要「自己選一位」
- 加一行「LINE push 預設開，可在後台關閉」，安撫黃品璇的「LINE 訊息過多」顧慮

### 給 Mentor 第二波（spec §13 Phase 3）

- 林思婕、黃品璇、吳沛文都直接或間接表達了「沒有女性 / 亞洲視角」的問題
- Phase 3 第二波 mentor 規劃時，**至少加 1 位女性 mentor**（候選：Tan Hooi Ling、Brené Brown 思想框架、雪柔·桑德伯格框架）
- 這不是政治正確的考量，是「目標客群有 60% 為女性」的產品設計需求

### 給流失預警（spec §12.2）

- **黃品璇型**：onboarding 完成率最危險，要監控 Step 4-5 跳出率
- **吳沛文型**：訂閱 3 個月內若 ROAS 低於 1.5，需要主動 retention 介入
- **張凱恩型**：trial 第 1 週若沒有第一次 sprint-plan，極可能流失
