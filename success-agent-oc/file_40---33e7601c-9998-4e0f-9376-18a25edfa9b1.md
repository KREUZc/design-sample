# Next 夥伴 ｜ 客戶體驗旅程 (Customer Journeys)

## 0. 文件定位

本文件接續 [customer-personas.md](customer-personas.md)，把 5 個 persona 從「靜態圖像」展開成「12 階段動態旅程」，作為 [success-agent-service-spec.md](success-agent-service-spec.md) §5「使用者旅程」的具象化補充。

**用法**：
- §8.1 行銷文案、§8.3 LINE Welcome Flex、§8.4 客戶後台 UX 設計皆以這 5 條旅程為驗收劇本
- Phase 0 MVP 的 Playwright E2E 測試（spec §13）以「陳定遠」這條旅程為 happy path 主腳本
- v1 上線前的可用性測試（usability test）以本 5 條旅程 × 各取 1 位真實受測者執行

**範圍外**：
- 個別頁面 wireframe（屬 spec §15 開放議題）
- A/B 測試方案（屬 Phase 2 行銷投放規劃）
- 客服 SOP 細節（屬 spec §15 開放議題）

## 1. 共用旅程結構（12 階段）

每位 persona 的旅程都按以下 12 階段展開。**不是所有階段都會逐字寫滿**，而是聚焦該 persona 的「關鍵分岔點」。

| 階段 | 內容 |
|---|---|
| 1. **觸發事件**（Day −7 ─ 0） | 在哪一刻「決定要找解方」 |
| 2. **Awareness** | 怎麼第一次知道 Next 夥伴 |
| 3. **Consideration** | 商品頁 / FAQ / 比價的關鍵問題 |
| 4. **Conversion**（Day 0） | 跨過試用門檻的最後一句話 |
| 5. **Onboarding 7 步**（Day 0） | 哪一步順、哪一步卡 |
| 6. **LINE 綁定 + Welcome**（Day 0 - 1） | 首次 mentor 對話的感受 |
| 7. **Sprint #1**（Day 1 - 3） | plan → checkin → pre-brief → final-call → retro 全程 |
| 8. **Trial → Paid**（Day 7） | 第 7 天那個下午的決定 |
| 9. **Month 2-3** | streak、月度信、第一次想取消的時刻 |
| 10. **推薦或流失**（Sprint #N） | 主動分享 vs 主動退訂的分岔 |
| 11. **期望體驗清單** | 給設計團隊的 punch list |
| 12. **對應 spec 段落** | 哪些 spec 條款被驗收 |

## 2. 共用前提

下列假設適用 5 條旅程，不在各章節重述：

- 用戶觸發旅程的時間點皆為 **產品 v1 公開上線後 1-3 個月**（spec §13 Phase 1 完成後）
- 起算日 Day 0 = 第一次造訪商品頁的當天
- 「試用 → 付費」第 7 天的決定，皆假設 Stripe 信用卡綁定當日已完成（spec §3.1）
- 所有 LINE 訊息延遲、推播時間、後台載入體驗皆假設達 spec §7.5 / §10.6 的目標 SLA
- mentor 真實性聲明（spec §1.4 / §11.4）已在所有 mentor 視角產出開頭附上 ─ 旅程描述不重複此聲明，但設計實作必須遵守

## 3. Journey 1 ─ 陳定遠 (SaaS Indie)

> **旅程定位**：典型「高意向、低摩擦」的理想型旅程。是 Phase 0 MVP Playwright E2E 主腳本的依據。

### 3.1 觸發事件 (Day −7 ─ 0)

**Day −7 週六晚上 23:30**：他打開 VS Code 看 Project C，6 週沒進展。git log 最後一筆 commit 是 6 週前。他在 LinkedIn 滑到一個前同事昨天剛 launch 第三個 SaaS，已經有 12 個付費客戶。

> 內心話：「我技術不比他差，他用的工具我都用過。差的是『有沒有人盯』這件事。」

**Day −1 通勤上**：他在 Twitter 看到一則貼文：「我把 Naval 的『永久遊戲』和 Musk 的『砍 90%』灌成一個 AI 同事，2 天問你一次 sprint 進度。NT$ 599/月。」貼文配一張看似 LINE Bot 對話的截圖。他先按讚，回家再看。

### 3.2 Awareness

**Day 0 21:00**：他坐下來翻 Twitter 找到那則貼文，點連結進 `success-agent.josephnext.com`。

**第一印象（Hero 區）**：
- 主標「2 天一個 sprint。Mentor 看著。Streak 連起來。」── 這就是他要的
- Demo 影片 30 秒：截 LINE 對話 + Mentor email 範例。**他看完整支** ── 這在他平常的 attention span 裡是極高轉換訊號

**滑到 Mentor 區**：4 張卡片，每張上面有真實性聲明。他點開 Naval 卡片，看到「適用議題：通路選擇、商業槓桿、財富建構」。他笑了 ─ 「正是我目前該想的事」。

### 3.3 Consideration

**他下拉到 FAQ**，點開三題：
1. 「跟 ChatGPT Plus 有什麼不同？」── 答案強調「主動推進 + mentor 視角 + 長期記憶 + 客戶後台可看可編輯」。他覺得買到的是「機制」不是「對話」
2. 「7 天試用後一定要付費嗎？」── 試用結束後首月 7 天內可退款。**這條讓他下定決心**
3. 「Mentor 真的是 Naval / Jobs 嗎？」── 真實性聲明清楚說「思想框架人格化模擬」。他可以接受

**他不打開的 FAQ**：取消政策、退款細節、隱私 ── 因為他現在還沒到那一步。**設計含義**：商品頁 FAQ 的展開順序要把「跟 ChatGPT 差別」「試用退款」放在前 3 題。

### 3.4 Conversion (Day 0, 21:18)

點「7 天免費試用」CTA → 註冊頁 → 輸入 Gmail → 收到 Magic Link → 點連結回網站 → 建立 session。整個流程 < 90 秒。

**Stripe Checkout**：選 Pro NT$ 599/月，trial 7 天。輸入信用卡，點「啟動試用」。Stripe 顯示「2026-05-07 將首次扣款」。**這個確切日期讓他安心**（不是「7 天後」這種模糊說法）。

> 內心話：「7 天後再決定，沒風險。」

### 3.5 Onboarding 7 步 (Day 0, 21:25 ─ 21:42)

| Step | 他的反應 |
|---|---|
| 1. 你是什麼人 | 選「SaaS 獨立開發者」+ 寫「外商工程師、副業在做 Project C」。**順** |
| 2. 想推進的事 | 一句話：「Project C launch 並收到第一個付費客戶」。詳細描述他打了 4 行。**順** |
| 3. Launch Definition | 系統提示：客戶語言版 + Agent 識別版。他寫「Stripe 收到第一筆 USD 付費 + Twitter 發 launch 貼文」。**順** |
| 4. Money Model | 「USD 19/月 SaaS、目標 100 客 = USD 1,900 MRR」。**極順**（他熟悉這格式） |
| 5. Avatar Map | 「marketing team 中的成長負責人，35 ─ 45 歲」── 他發現自己只能寫一行。**這是第一個摩擦點，但不致命** |
| 6. sprint 偏好 | 預設 2 天 21:00。**他改成 23:59** ─ 「壓底族」 |
| 7. 風格 + email | cold + 個人 Gmail。**他遲疑 5 秒選 cold**（直覺以為 cold 會比較刺，但他要的就是這個） |

22 分鐘完成。Worker 後台同時建立 `clients/{user_id}/` 5 檔骨架。

### 3.6 LINE 綁定 + Welcome (Day 0, 21:43)

點「綁定 LINE」→ LINE Login OAuth → 授權 → 回 callback → 頁面顯示「加好友」QR Code + 連結。他在手機點連結 → LINE 開啟「Next 夥伴」官方帳號 → 加好友 → 收到 Welcome Flex Message。

**Welcome 訊息他看到的**：
- 卡片頭：「Next 夥伴 / 你專屬的客戶成功經理」
- 4 位 mentor 縮圖（Jobs / Buffett / Naval / Musk），下方一句「指派由 AI 視議題自動」
- 一句：「準備好就傳訊息給我。第一個 sprint 我們從你的 Launch Definition 起步。」
- 真實性聲明縮寫版

> 內心話：「設計感比我預期好。」

他立刻打字：「我們開始」 → Agent 觸發 health-check skill → 回覆是 Naval 視角（Agent 自動指派，因為 Money Model 屬槓桿議題） → **首句是真實性聲明**，第二段才是健診結果。

> 內心話：「OK，這聲明放第一段我可以接受，免得每段都有讓我煩。」

### 3.7 Sprint #1 (Day 1 ─ 3)

**Day 1 早上 09:00 (sprint-plan)**：他通勤捷運上跟 Agent 對話。Agent 用 Musk 視角（議題從 Naval 切到 Musk，因為他想「砍 Project C 範圍 launch MVP」）：

> 「砍掉 90% 的功能，剩下的 10% 是哪些？2 天能 ship 的最小可付費版本長什麼樣？」

他回了 4 個功能，Agent 反問「真的都需要？」。剔到 2 個。承諾任務寫進 sprint.md。

**Day 1 中午**：Agent 不主動推。他午餐後傳「我寫完 task 1 的一半」。Agent 用 Musk 視角回「半完成不算 done。今天 23:59 前 ship task 1，剩下時間不夠 task 2，task 2 留下個 sprint」。

**Day 2 23:00 (T - 12h pre-brief email)**：他收到 mentor 預讀 email。標題「Sprint #1 retro 預讀 / Musk 視角」。內容是 Musk 對他這次 sprint 的觀察 + 4 個提問。**他在床上看完，截圖傳給太太「這個 AI 寫得有點意思」**。

**Day 3 22:00 (T - 2h final-call email)**：「最後 2 小時。task 2 還沒完成。最小可完成版本：直接寫 README 描述 task 2 預期功能，commit 進 repo，下個 sprint 接手。」── 他立刻照做。**這封 email 是他試用第 3 天最有感的單一動作。**

**Day 3 23:59 retro**：Agent 已預讀，直接從「為什麼 task 2 沒做完」起手。Streak 算法判定「task 1 done，task 2 carry」── streak 不中斷。Agent 結尾：「Sprint #1 closed. 1 done / 1 carry. streak: 1.」

> 內心話：「就這樣，第一個 sprint 跑完。」

### 3.8 Trial → Paid (Day 7)

Day 4-6 跑完 Sprint #2、Sprint #3 開始。

**Day 7 中午**：他收到 Resend 系統信「明天首次扣款 NT$ 599」。他登入 `/account/billing` 看下次扣款日確認 ─ 沒退訂。

> 內心話：「3 個 sprint 跑下來，比過去 6 週還多。值得」

**他甚至已經想下一步**：續訂繼續用 Project C；下個月若 launch 了，會考慮跟 Twitter 朋友推薦這工具。

### 3.9 Month 2-3

- Sprint streak 在 Sprint #5 第一次中斷（出差），他截圖傳 Twitter「streak 0 重啟」 ─ **這是 Phase 1 streak-celebration 機制的副產品**
- Sprint #8 結束時 Project C 到 closed beta，14 個試用客戶
- Month 2 第 1 號收到月度 mentor 信：Musk 視角，回顧本月 sprints。**他全文讀完，轉發給太太**
- Month 3 第 2 週：考慮要不要再多開一個 product 用同 Agent ─ 後台「Agent 對我的整體理解」按鈕讓他確認 Agent 知道他的 context，**他決定繼續**

### 3.10 推薦或流失

**Sprint #12 結束**（約 Month 3 末）：Project C 收到第一個付費客戶。他在 Twitter 發貼文，**自然提到 Next 夥伴**：

> 「過去 3 個月用 NT$ 599/月的 AI mentor 跑 sprint。streak 9。launch 了 Project C。不是 affiliate。`success-agent.josephnext.com`」

這條推文預期帶來 5-15 個高意向流量 ─ **獲客槓桿**。

**流失情境（10% 機率）**：Project C 第 2 個月因 PMF 不對決定收掉。他短暫想退訂，但 Agent 引導他「下個 project 從這個記憶面板繼續」 → 他啟動 Project D，未流失。

### 3.11 期望體驗清單

- [ ] Stripe Checkout 顯示**確切日期**（非「7 天後」），spec §3.1 已對齊
- [ ] Onboarding Step 5 Avatar Map 對「他熟悉但只能寫一行」的情況不要逼他寫滿 ── 提示文案「先寫一行也可以，後面 Agent 會問你補」
- [ ] LINE Welcome 真實性聲明放第一段且**支援摺疊**（避免每次對話都看到全文）
- [ ] sprint-final-call email 必須在預期時間 ± 30 分送達（spec §7.3 排程）
- [ ] 月度 mentor 信品質要值得截圖轉發 ── Resend 模板設計時必試讀

### 3.12 對應 spec 段落

- spec §5 使用者旅程整條（這是最完整命中的 happy path）
- spec §3.3 Onboarding 7 步（Step 5 摩擦對應 §15 開放議題）
- spec §7.3 三類排程任務（pre-brief / final-call / monthly-letter）皆驗收
- spec §8.4.2 記憶面板「Agent 對我的整體理解」按鈕

## 4. Journey 2 ─ 林思婕 (知識變現創作者)

> **旅程定位**：高黏性、高推薦力，但 onboarding 中段會自我懷疑「我適合 sprint 嗎」。是 retention design 的關鍵驗證。

### 4.1 觸發事件 (Day −10 ─ 0)

**Day −10 週日晚上**：她該寫 Substack 那期專欄但寫不出來。她在 IG 滑到一個朋友（也是創作者）發限動：「這個月我寫了 5 篇電子報，前所未有。秘訣是有人在中午跟我說『你昨天說要寫的還沒交』」。她截圖私訊問。

**Day −3**：朋友回覆，附產品截圖。她開始追蹤產品 IG 帳號 + 看 Demo 影片。

**Day 0 早上 10:30**：她剛回完客戶 LINE，泡完第二杯咖啡，決定點開連結。

### 4.2 Awareness

她進 `success-agent.josephnext.com`。Hero 主標「2 天一個 sprint」── **她皺眉**。

> 內心話：「2 天太短了，創作不是這樣。」

她差點關掉。但 Demo 影片下方的 sub-hook（[customer-personas.md](customer-personas.md) §8 衍生建議）「也支援 1 週 sprint」讓她滑下去。

她看 Mentor 卡片：Jobs ─「適用議題：UI/UX 決策、功能取捨、發布品質」。她想：「『發布品質』── 這對我寫文章是相關的。」

### 4.3 Consideration

她展開 5 個 FAQ：
1. 「我不熟 sprint 怎麼辦？」── 答案：「Agent 在 onboarding 教你」。她安心
2. 「跟 ChatGPT Plus 差別？」── 強調記憶面板 / 主動推進 / mentor 視角
3. 「Mentor 是真的 Jobs 嗎？」── 真實性聲明 + 引用來源說明。她接受
4. 「可以中途取消嗎？」── 隨時可以、首月 7 天內全退
5. 「我的內容 / 創作思路會被儲存嗎？」── 個資政策 + 「不記憶模式」（v1.5）。她對 v1.5 模式有期待，但 v1 就先接受

她滑到頁面底「見證 / 案例」區看到 Fred 的 founder story。**她對「自己也是創作者」的這個敘事極有共鳴**。

### 4.4 Conversion (Day 0, 11:05)

她註冊 + 試用。Magic Link 4 分鐘後才收到（Resend 投遞）── 她差點關頁面。Magic Link 進信箱後她點連結回來繼續。

> 設計含義：Resend 寄送速度必須 < 60 秒到達，60 秒以上的延遲會吃掉這群「容易分心」的用戶

### 4.5 Onboarding 7 步 (Day 0, 11:09 ─ 11:32)

| Step | 她的反應 |
|---|---|
| 1. 你是什麼人 | 選「全職個人品牌經營者 / 內容創作者」+ 描述 3 行。**順** |
| 2. 想推進的事 | 「線上課第二期招生 + Substack 從雙週轉週更」。她寫了兩件事，但 Step 2 設計只接一件 ── **這是摩擦點**。她選「Substack 週更」 |
| 3. Launch Definition | 「Substack 連續 8 週每週準時出刊」。**她遲疑** ─「Launch 對我不是『一次發布』，是『持續產出』」。她最後填「8 週準時出刊算 launch 一次」 |
| 4. Money Model | 「Substack USD 5/月 × 320 付費 = USD 1,600/月。目標 1 年內到 USD 5,000/月」。**順** |
| 5. Avatar Map | 「30-40 歲女性、職涯轉換期、有意識想經營個人品牌」。**順**（這是她日常做的事） |
| 6. sprint 偏好 | 預設 2 天 21:00。**她改成 1 週 21:00**（她直覺 2 天太短） |
| 7. 風格 + email | passionate + 個人 Gmail。**她選 passionate**（不想被太硬刺到） |

23 分鐘完成。**Step 2「只能填一件事」是這趟 onboarding 最大摩擦**，需要 v1.5 改進為「主任務 + 支線任務」二欄。

### 4.6 LINE 綁定 + Welcome (Day 0, 11:33)

LINE 綁定 + 加好友。Welcome 訊息她讀完真實性聲明 ── 接受度高。

**首次對話**：她傳「你好」 → Agent 觸發 health-check（首次健診）。Agent 用 Buffett 視角（因為她剛填的 Money Model 是議題重心）：

> 「30 秒思考：你的 Substack 護城河是什麼？你看到的長期遊戲，跟你目前每週實際在做的事，對齊嗎？」

她愣了一下，回了 80 字。Agent 把這段重要敘述標記為 `memories`（type: business_context, importance: 5），同步寫進 memories 表。

> 內心話：「它真的在記。我在後台應該看得到。」── 她立刻打開 `/account/memory`，看到這條 memory，編輯了一個錯字。**這個動作她記得很久**。

### 4.7 Sprint #1 (Day 1 - 7)

**Day 1 早上**：sprint-plan。她列了 4 件事：寫一篇 Substack、設計線上課宣傳、回 5 個學員 email、IG 發 3 篇。

Agent 用 Jobs 視角（議題切換到「內容品質」）：

> 「4 件事都做完，但每件都七分。或一件做到九分，其他改下個 sprint。你選哪個？」

她想了 3 分鐘，回「一件九分 ─ Substack」。其他三件挪到下個 sprint。

**Day 3-5**：她每天傳 1-2 則對話，多半是「我卡在這段」「這個標題改成 X 怎樣」。Agent 在內容卡關時 Jobs 視角介入，其他時候只用 base passionate 回應。**她感覺 mentor 不會「亂介入」很重要**。

**Day 6 早上 9:00 (T - 12h pre-brief)**：她收到 Jobs 視角的 retro 預讀 email。**她截圖發 IG 限動**：「我在用一個 AI 同事，這是它今早預讀我這週 sprint 的觀察。」 ─ 限動 12 小時內 800 人看，5 人私訊問是什麼產品。

**Day 6 19:00 (T - 2h final-call)**：她收到 final-call。她那天剛好沒寫完，但讀完 email 中的「最小可完成版本」（800 字寫不完先寫 400 字 + 一張圖），她決定衝。

**Day 6 21:00 retro**：Sprint #1 closed. 1 done / 0 carry. streak: 1. **她截圖**。

### 4.8 Trial → Paid (Day 7)

Day 7 早上她收到「明天首次扣款」通知。她沒退訂。

**真正的決定點是 Day 6 那則 IG 限動**。當她發限動的那一刻，她已經把 Next 夥伴納入她的「個人品牌敘事」。退訂等於否定那個敘事。

### 4.9 Month 2-3

3. **「我是接案 + 教學型，這套適合嗎？」** ─ FAQ 必須有這條（[customer-personas.md](customer-personas.md) §8 衍生建議）
4. 「LINE Bot 會不會吵？」── 可關
5. 「能取消嗎？」── 是

**FAQ #3 是她唯一明確的「我是不是被涵蓋」確認**。如果這條不存在，她會關掉。

### 7.4 Conversion (Day 0, 7:55)

註冊 + 試用。她猶豫 4 分鐘綁信用卡。最後因為「7 天免費 + 首月退款」+「老公會 OK」（她實際思考了配偶反應）下決定。

### 7.5 Onboarding 7 步 (Day 0, 8:00 ─ 8:38)

| Step | 她的反應 |
|---|---|
| 1. 你是什麼人 | 「自由接案 UI 設計師 + 線上設計小課」 |
| 2. 想推進的事 | 「線上課第三期擴大到 50 人 + 開始寫設計師 newsletter」**她又卡兩件事**（同林思婕、吳沛文） |
| 3. Launch Definition | 「第三期線上課 50 人額滿」**她寫得快** |
| 4. **Money Model** ─ **她在這步卡 8 分鐘** | 「接案月 NT$ 100,000-130,000、線上課每期 NT$ 100,000-150,000」**她不確定這算不算「Money Model」**。她差點離開。**最後她寫了「我不確定這格式對不對，但這是我的收入結構」** |
| 5. Avatar Map | 「30-45 歲設計師、想開課但卡在規劃」**順**（她每天面對這群） |
| 6. sprint 偏好 | **她改 1 週 21:00** |
| 7. 風格 + email | passionate + Gmail。**她猶豫要不要改成另一個 email** ─ 怕跟接案 email 混（最後選了同一個） |

38 分鐘完成。**Step 4 是這趟 onboarding 的最大關卡**。系統若沒有「接案 / 教學型範本」，黃品璇有 30% 機率在此放棄。

### 7.6 LINE 綁定 + Welcome (Day 0, 8:40)

LINE 綁定。她**立刻到後台 `/account/persona` 把 LINE push 關掉**（同張凱恩，但理由不同 ─ 她怕被接案訊息淹沒）。

### 7.7 Sprint #1 (Day 1 - 7)

**Day 1 早上**：sprint-plan。Agent 觸發 health-check（首次健診）。Naval 視角：

> 「12 年接案，你的『時間』就是你的全部資產。線上課是你想用的『槓桿』。但你目前的『時間分配』，跟你想要的『槓桿轉移』，落差有多大？」

她敲鍵盤敲到一半哭了 ─ **這是她整個 onboarding + Sprint #1 中最有感的單一句子**。她回了 200 字。Agent 把這段標記為 memory（importance 5）。

**Day 1-5 平日**：她每天傳 1-2 則對話，多半是夜晚 22:00 後。Agent 不催她進度（她已說明白天要接案）。

**Day 6 21:00 (T - 12h pre-brief)**：她讀到 Naval 視角的預讀 email。**她哭了第二次**。**她截圖傳給她設計師朋友**。

**Day 7 19:00 (T - 2h final-call)**：3 件事完成 2 件。Agent 引導「最小可完成版本」。她照做。

**Day 7 21:00 retro**：Sprint #1 closed. 3 done / 0 carry. streak: 1. **她截圖收進她的「個人成長相簿」**。

### 7.8 Trial → Paid (Day 7)

Trial 結束日 = Day 7。**她沒退訂**。

> 內心話：「Naval 那句話我會記一年。NT$ 599 我已經回本了。」

### 7.9 Month 2-3

- Month 1 末她開始寫 newsletter（Sprint #3 完成第一篇）
- Month 2 第 1 號月度 mentor 信（Naval 視角）── **她全文讀完並列印出來貼書桌**
- Sprint #6 中斷一次（家裡長輩生病）。她想取消，但 Agent 偵測中斷理由（她在對話中說了原因），主動建議「暫停 1 個月、無扣款」 ─ **這個機制 spec 沒寫，是 v1.5 retention design 必加**
- Month 3 第 4 週她回來，繼續

### 7.10 推薦或流失

**推薦**：在女性創業 LINE 群、設計師 Slack、podcast 朋友間私下推薦。**不會公開貼文**（她不是 KOL）。這型推薦轉換率高（密群、同質性高）但量小。

**流失情境（35% 機率）**：家庭事件 + 接案旺季同時來 → 連 sprint 都放空 → 半年後想起取消。**設計含義**：v1.5 必加「主動暫停」機制，否則這型客戶會變成「殭屍訂閱」（沒用但也沒退），最後一次性退並負評。

### 7.11 期望體驗清單

- [ ] **Onboarding Step 4 Money Model 提供「接案 / 教學型」範本**（最高優先級，影響黃品璇型 onboarding 完成率 30%）
- [ ] FAQ 必須有「我是接案 / 教學型適合嗎？」專題
- [ ] 後台 `/account/persona` 的「LINE push 開關」要顯眼
- [ ] **「主動暫停 1 個月」機制（v1.5 加）**：偵測連續 2 sprint 完全空白 → 主動 email 詢問「需要暫停嗎，無扣款」
- [ ] 月度 mentor 信對她有「儀式感」 ─ Resend 模板要支援列印（A4 版面友善）

### 7.12 對應 spec 段落

- spec §3.3 Step 4 Money Model ── 必須擴充為**多型態範本**
- spec §8.4.3 風格與推播設定中「LINE push 開關」對她極重要
- spec §15 開放議題「Onboarding 7 步問卷的具體問句與引導文案」── 她是這個議題的關鍵測試對象
- spec §13 Phase 2「Lite 方案」── 暫停機制比 Lite 方案更需要被優先實作（她的需求不是「便宜」，是「彈性」）

## 9. 衍生設計需求（給 spec / 後續文件）

依據 5 條旅程的橫向觀察，以下需求**僅列項目**，不在本文件展開：

### 給 spec §3.3 Onboarding 7 步問卷

1. **Step 2「想推進的事」改為「主任務 + 最多 2 個支線」**（解林思婕 / 吳沛文 / 黃品璇都卡的問題）
2. **Step 3 Launch Definition 對「持續型 launch」提供範例**（解林思婕的「我的 launch 是連續產出」）
3. **Step 4 Money Model 提供 4 個範本**（電商 / 服務業 / 內容創作 / 諮詢），對黃品璇 / 吳沛文影響最大
4. **Step 6 sprint 預設 2 天，但 1 週按鈕同等顯眼**（張凱恩、林思婕、黃品璇都會改）
5. **Step 7 推播 email 加註「不會用於行銷」**（解張凱恩的隱私顧慮）

### 給 spec §8.1 商品頁

1. Hero 主標下方加 sub-hook「也支援 1 週 sprint」（解林思婕 / 黃品璇的初印象問題）
2. FAQ 必加 5 題：
   - 「我不熟 sprint 怎麼辦？」
   - 「公司資料 / 隱私如何處理？」
   - 「中文回應品質如何？」
   - 「我是接案 / 教學型適合嗎？」
   - 「LINE Bot 會不會吵？可不可以關？」
3. 見證 / 案例區須涵蓋 5 種角色類型（不只科技 SaaS）
4. 加入 IG / Twitter 連結作為 nurture 入口（解吳沛文「不當天決定」流失）

### 給 spec §7 Agent 整合層

1. **連續 2 sprint 未完成的自動降階**（對話風格從 mentor 視角降回 base passionate / cold）── 解林思婕「越嚴越逃」風險
2. **主動暫停 1 個月機制**（偵測連續 2 sprint 完全空白 → email 詢問）── 解黃品璇「家庭+旺季」流失
3. **首次健診 health-check 對 5 種角色類型有客製問句**（不只通用 21 題）

### 給 spec §10.4 Resend 模板

1. **pre-brief / final-call email 設計上「截圖友善」**（林思婕、吳沛文、黃品璇都會截圖）
2. **月度 mentor 信支援列印 A4 版面**（黃品璇）
3. **Magic Link 投遞速度 < 60 秒**（林思婕容易在等待時分心）

### 給 spec §12.4 行銷漏斗追蹤

1. 流量來源分 4 類：「KOL 自然推薦」「私推（無連結）」「直接流量」「付費投放」
2. 「KOL 自然推薦」需獨立追蹤 referer header
3. 「私推（無連結）」靠 onboarding Step 0「你怎麼知道我們的？」自報

### 給 spec §15 開放議題

1. **Onboarding 7 步問卷具體問句**：本文件 §3-7 的 Step-by-step 是規格起點
2. **LINE Flex Message 設計範本**：本文件 §3.6 / §6.6 的 Welcome 體驗描述是設計起點
3. **訂閱續扣失敗 retry / dunning 策略**：本文件 §5.10（張凱恩）+ §6.10（吳沛文）+ §7.10（黃品璇）的流失情境是策略設計起點
